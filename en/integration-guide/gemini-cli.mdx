## Overview

![Gemini CLI Interface](/images/integration-guide/image-2.png)

Gemini CLI is Google's official command-line tool for interacting with Gemini models in the terminal.
By integrating Gemini CLI with **EvoLink API**, you can go beyond official model limits and access EvoLink's broader model set, including **Gemini, GPT, and Claude** series models, through one entry point.

## Prerequisites

Before configuring, make sure you have:

### 1. Install Node.js and npm

- Download and install from [Node.js official website](https://nodejs.org/)
- Recommended: **Node.js v16 or higher**
- Verify installation:

```bash
node -v
npm -v
```

### 2. Get EvoLink API Key

- Log in to [EvoLink Dashboard](https://evolink.ai/dashboard)
- Create and copy your API Key
- API Key usually starts with `sk-`

## Step 1: Install Gemini CLI

```bash
npm install -g @google/gemini-cli
```

Installation successful:

![Gemini CLI Install](/images/integration-guide/image.png)

### 2. Verify Installation

```bash
gemini --version
```

## Step 2: Configure EvoLink API

### 1. Find installation directory

```bash
npm root -g
```

### 2. Modify API endpoint

Modify 2 files:

**File 1:** `{install_dir}\\@google\\gemini-cli\\node_modules\\@google\\genai\\dist\\node\\index.mjs`

Find line **~11222**:

```javascript
// Before
initHttpOptions.baseUrl = `https://generativelanguage.googleapis.com/`;

// After
initHttpOptions.baseUrl = `https://api.evolink.ai/`;
```

**File 2:** `{install_dir}\\@google\\gemini-cli\\node_modules\\@google\\genai\\dist\\node\\index.cjs`

Find line **~11244**, make the same change.

### 3. Modify default model (optional)

**File:** `{install_dir}\\@google\\gemini-cli\\node_modules\\@google\\gemini-cli-core\\dist\\src\\config\\defaultModelConfigs.js`

Find line **~96-99**:

```javascript
// Before
classifier: {
    extends: 'base',
    modelConfig: {
        model: 'gemini-2.5-flash-lite',

// After
classifier: {
    extends: 'base',
    modelConfig: {
        model: 'gemini-2.5-flash',  // Use a model supported by EvoLink
```

> **Note**: Only change this if EvoLink does not support `gemini-2.5-flash-lite`.

### 4. Configure API Key

**Option 1: Temporary (current session only)**

```powershell
$env:GEMINI_API_KEY = "your-evolink-api-key"
```

**Option 2: Permanent**

```powershell
[Environment]::SetEnvironmentVariable("GEMINI_API_KEY", "your-evolink-api-key", "User")
```

## Step 3: Start Using Gemini CLI

### 1. Enter a safe working directory

```powershell
cd D:\mycode\GeminiTest
```

### 2. Single query mode

```shell
gemini "Who are you"
```

### 3. Interactive mode

```shell
gemini
```

If you receive a normal response, the configuration is successful.

## FAQ

### 1. How do I switch models?
- Type `/model` in the prompt

![Gemini CLI Switch Model](/images/integration-guide/image-3.png)

- Available models:
  - `gemini-2.5-pro`
  - `gemini-2.5-flash`
  - `gemini-3-pro-preview`
  - `gemini-3-flash-preview`

### 2. How do I upload images?
- Option 1: Reference the image path
- Option 2: Drag and drop an image into the terminal

## Notes

<Warning>
Run Gemini CLI in a safe directory. It can only read files in the current working directory.
