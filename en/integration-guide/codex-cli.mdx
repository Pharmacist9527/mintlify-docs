## Overview

Codex CLI is OpenAI's official command-line tool for code-related tasks in the terminal.
By integrating Codex CLI with **EvoLink API**, you can use EvoLink's models through a unified OpenAI-compatible interface.

## Prerequisites

### 1. Install Node.js and npm

- Download from [Node.js official website](https://nodejs.org/)
- Recommended: **Node.js v18 or higher**
- Verify installation:

```bash
node -v
npm -v
```

### 2. Get EvoLink API Key

- Log in to [EvoLink Dashboard](https://evolink.ai/dashboard)
- Create and copy your API Key
- API Key usually starts with `sk-`

## Step 1: Install Codex CLI

### 1. Global Installation

```bash
npm install -g @openai/codex
```

### 2. Verify Installation

```bash
codex --version
```

## Step 2: Configure EvoLink API

Codex CLI supports custom Provider via config file, no source code modification needed.

### 1. Find Config File

Config file location:

```
C:\Users\{username}\.codex\config.toml
```

### 2. Edit Config File

Edit `config.toml` with the following content:

```toml
model = "gpt-5.2"
model_reasoning_effort = "medium"
model_provider = "evolink"

[model_providers.evolink]
name = "EvoLink API"
base_url = "https://api.evolink.ai/v1"
env_key = "OPENAI_API_KEY"
wire_api = "responses"
```

### 3. Configure API Key

**Option 1: Temporary (current session only)**

```powershell
$env:OPENAI_API_KEY = "your-evolink-api-key"
```

**Option 2: Permanent**

```powershell
[Environment]::SetEnvironmentVariable("OPENAI_API_KEY", "your-evolink-api-key", "User")
```

### 4. Supported Models

| Model Name | Description |
|------------|-------------|
| `gpt-5.1` | GPT 5.1 |
| `gpt-5.2` | GPT 5.2 |

## Step 3: Start Using Codex CLI

### 1. Enter a safe working directory

```powershell
cd D:\mycode\CodexTest
```

### 2. Interactive mode

```shell
codex
```

### 3. Single query mode

```shell
codex "Who are you"
```

## Notes

<Warning>
Run Codex CLI in a safe directory. It can only read files in the current working directory.
</Warning>

<Tip>
The `wire_api` in config must be set to `"responses"`. `"chat"` is deprecated.
</Tip>
