## Overview

Claude CLI is the official command-line tool from Anthropic for interacting with Claude models in the terminal.
By integrating Claude CLI with **EvoLink API**, you can directly access Claude model capabilities through EvoLink.

## Prerequisites

### 1. Install Node.js and npm

- Download and install from [Node.js official website](https://nodejs.org/)
- Recommended: **Node.js v18 or higher**
- Verify installation:

```bash
node -v
npm -v
```

### 2. Get EvoLink API Key

- Log in to [EvoLink Dashboard](https://evolink.ai/dashboard)
- Create and copy your API Key
- API Key usually starts with `sk-`

## Step 1: Install Claude CLI

### 1. Global Installation

```bash
npm install -g @anthropic-ai/claude-code
```

### 2. Verify Installation

```bash
claude --version
```

## Step 2: Configure EvoLink API

Claude CLI supports API endpoint configuration via config file, no source code modification needed.

### 1. Find Config File

Config file location:

```
C:\Users\{username}\.claude\settings.json
```

### 2. Edit Config File

Edit `settings.json` with the following content:

```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "your-evolink-api-key",
    "ANTHROPIC_BASE_URL": "https://api.evolink.ai",
    "CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC": "1"
  },
  "permissions": {
    "allow": [],
    "deny": []
  },
  "model": "claude-sonnet-4-5-20250929"
}
```

### 3. Supported Models

| Model Name | Description |
|------------|-------------|
| `claude-haiku-4-5-20251001` | Fast response |
| `claude-sonnet-4-5-20250929` | Balanced |
| `claude-sonnet-4-20250514` | Standard |
| `claude-opus-4-1-20250805` | Advanced |
| `claude-opus-4-5-20251101` | Latest |

## Step 3: Start Using Claude CLI

### 1. Enter a safe working directory

```powershell
cd D:\mycode\ClaudeTest
```

### 2. Interactive mode

```shell
claude
```

### 3. Single query mode

```shell
claude "Who are you"
```

## Notes

<Warning>
Run Claude CLI in a safe directory. It can only read files in the current working directory.
</Warning>

<Tip>
If you previously logged in with an official account, clear the `ANTHROPIC_AUTH_TOKEN` environment variable or override it in `settings.json`.
</Tip>
