## 概要

OpenClaw は、チャットアプリケーションと AI エージェント間の橋渡しとして機能するオープンソースの AI エージェント Gateway です。集中型の Gateway プロセスを通じて、Telegram、WhatsApp、Discord、Feishu などのチャットプラットフォームを AI プログラミングエージェントに接続できます。

このドキュメントでは、OpenClaw を手動でインストールし、**EvoLink API** をモデルプロバイダーとして設定する方法について説明します。このドキュメントを完了した後、特定のチャットチャネル（Telegram や Feishu など）の設定を続けることができます。

**このガイドの内容:**
- OpenClaw Gateway のインストールと設定
- EvoLink API をカスタムモデルプロバイダーとして設定
- インストールの確認

## システム環境チェック（オプション）

インストールを開始する前に、環境チェックツールを実行して、システムが OpenClaw の要件を満たしていることを確認することをお勧めします。

### チェックツールのダウンロード

[GitHub Releases](https://github.com/suuuuuu-1/openclaw-env-checker/releases) からプラットフォームに対応したチェックツールをダウンロードしてください：

| プラットフォーム | ファイル名 |
|-----------------|-----------|
| Windows | `openclaw-checker-win-x64.exe` |
| macOS (Intel) | `openclaw-checker-macos-x64` |
| macOS (Apple Silicon) | `openclaw-checker-macos-arm64` |
| Linux | `openclaw-checker-linux-x64` |

### チェック項目

ツールは以下を自動的にチェックします：

- ✅ Node.js バージョン（>= 22.12.0 が必要）
- ✅ npm が利用可能
- ✅ Git が利用可能
- ✅ ネットワーク接続（github.com、npmjs.org、evolink.ai）

![チェック成功例](/images/OpenClaw-Checker/check-success.png)

チェックが失敗した場合、ツールは具体的な修正提案を提供します。

## 前提条件

設定を開始する前に、以下を完了していることを確認してください：

### 1. Node.js のインストール

OpenClaw は npm 経由でインストールされ、Node.js 22 以上が必要です。

<Tabs>
  <Tab title="Windows">
    [Node.js 公式サイト](https://nodejs.org/ja/download)にアクセスし、Windows インストーラー（.msi ファイル）をダウンロードして実行します。

    インストール後、PowerShell で確認します：

    ```bash
    node --version
    npm --version
    ```

    <Note>
    インストール中の権限の問題を避けるため、PowerShell を管理者として実行することをお勧めします。
    </Note>
  </Tab>
  <Tab title="macOS">
    **方法 1: インストーラーを使用**

    [Node.js 公式サイト](https://nodejs.org/ja/download)にアクセスし、macOS インストーラー（.pkg ファイル）をダウンロードして実行します。

    **方法 2: Homebrew を使用**

    ```bash
    brew install node
    ```

    インストール後、ターミナルで確認します：

    ```bash
    node --version
    npm --version
    ```

    <Note>
    インストール中に権限の問題が発生した場合、コマンドの前に `sudo` を追加する必要があるかもしれません。
    </Note>
  </Tab>
</Tabs>

### 2. EvoLink API キーの取得

- [EvoLink コンソール](https://evolink.ai/dashboard)にログインします
- コンソールで API Keys を見つけ、「新しいキーを作成」をクリックして、生成されたキーをコピーします
- API キーは通常 `sk-` で始まります。安全に保管してください

## ステップ 1: OpenClaw のインストール

ターミナルで実行します：

```bash
npm install -g openclaw@latest
```

インストール後に確認します：

```bash
openclaw --version
```

## ステップ 2: 初期設定

オンボーディングコマンドを実行します。OpenClaw が初期設定をガイドし、デーモンサービスをインストールします：

```bash
openclaw onboard --install-daemon
```

## ステップ 3: EvoLink API とモデル切り替えの設定

### 1. 2つの設定ファイルを見つける（重要）

OpenClaw のモデル設定には通常、2つのファイルが関係します：

<Tabs>
  <Tab title="Windows">
    - `openclaw.json`: `%USERPROFILE%\.openclaw\openclaw.json`
    - `models.json`: `%USERPROFILE%\.openclaw\agents\main\agent\models.json`
  </Tab>
  <Tab title="macOS">
    - `openclaw.json`: `~/.openclaw/openclaw.json`
    - `models.json`: `~/.openclaw/agents/main/agent/models.json`

    直接開く：

    ```bash
    open ~/.openclaw/openclaw.json
    open ~/.openclaw/agents/main/agent/models.json
    ```
  </Tab>
</Tabs>

<Warning>
`models.json` 内のプロバイダーの `apiKey` / `baseUrl` が空でない場合、`openclaw.json` の対応する値を上書きします。両方を一致させることをお勧めします。
</Warning>

### 3. デフォルトモデルの設定（クイック切り替えをサポート）

`agents.defaults` でデフォルトモデルを設定します：

```json
"agents": {
  "defaults": {
    "model": {
      "primary": "evolink-anthropic/claude-opus-4-6"
    }
  }
}
```

一般的な切り替え例：

- GPT モデルに切り替え: `evolink-openai/gpt-5.2`
- Gemini に切り替え: `evolink-google/gemini-3.1-pro-preview`
- Claude に切り替え: `evolink-anthropic/claude-opus-4-6`
- Doubao に切り替え: `evolink-openai/doubao-seed-2.0-mini`

### 4. EvoLink モデルへのクイック切り替え（推奨）

プロバイダー設定が完了したら、JSON を手動で編集する代わりに CLI コマンドを使用してモデルを切り替えることをお勧めします：

```bash
# 設定された EvoLink OpenAI 互換モデルを表示
openclaw models list --provider evolink-openai --plain

# デフォルトモデルを切り替え（例: gpt-5.4）
openclaw models set evolink-openai/gpt-5.4

# 現在のアクティブなモデルを表示
openclaw models status --plain
```

<Note>
`models list --provider evolink-openai` で期待するモデルが表示されない場合、`openclaw.json` と `models.json` の両方に対応するプロバイダーが設定されているか確認してください。
</Note>

### 5. 再起動と確認

設定後にゲートウェイを再起動します：

```bash
openclaw gateway restart
```

ステータスを確認します：

```bash
openclaw gateway status
```

テストメッセージを送信してモデルが動作していることを確認します：

```bash
openclaw agent --agent main -m "hi" --json
```

## よく使うコマンド

| コマンド | 説明 |
|---------|------|
| `openclaw gateway status` | ゲートウェイの実行状態を確認 |
| `openclaw gateway restart` | ゲートウェイサービスを再起動 |
| `openclaw gateway stop` | ゲートウェイサービスを停止 |
| `openclaw gateway start` | ゲートウェイサービスを開始 |
| `openclaw logs --follow` | ゲートウェイログをリアルタイムで表示 |
| `openclaw plugins list` | インストール済みプラグインを表示 |

## トラブルシューティング

| 問題 | 解決方法 |
|------|---------|
| npm インストールが失敗する | Windows: PowerShell を管理者として実行; macOS: コマンドの前に `sudo` を追加 |
| 設定ファイルが見つからない | オンボーディングプロセスが完了していることを確認し、`~/.openclaw/` ディレクトリが存在するか確認 |
| ゲートウェイが起動しない | ポートが占有されているか確認し、`openclaw gateway status` で詳細なエラーを表示 |
| 無効な API キー | API キーが正しくコピーされていることを確認し、余分なスペースや引用符がないか確認 |
| モデル設定が有効にならない | `openclaw.json` と `models.json` の両方の整合性を確認（`models.json` が上書きする可能性があります） |
| Gemini が `Forbidden (403)` を返す | `models.providers.evolink-google.baseUrl` が `https://direct.evolink.ai/v1beta` であることを確認（`/v1beta` を含める必要があります） |
| 切り替え後も古いモデルが使用される | `openclaw models status --plain` を実行して現在のモデルを確認し、必要に応じて `openclaw gateway restart` で再起動 |
