## Aperçu des paramètres

| Paramètre | Syntaxe | Type | Plage | Par défaut | Description |
|-----------|---------|------|-------|------------|-------------|
| Rapport d'aspect | `--ar W:H` | Ratio entier | Tout ratio d'entiers positifs | 1:1 | Rapport d'aspect de l'image |
| Chaos | `--c N` | int | 0 - 100 | 0 | Diversité des résultats générés |
| Prompt négatif | `--no mot1,mot2` | Mots-clés | — | — | Exclure les éléments indésirables |
| Qualité | `--q N` | Enum | 1, 2, 4 | 1 | Niveau de détail de la génération, **affecte le coût** |
| Seed | `--seed N` | int | 0 - 4294967295 | Aléatoire | Fixer le seed pour des résultats reproductibles |
| Stylisation | `--s N` | int | 0 - 1000 | 100 | Intensité du style artistique |
| Weird | `--w N` | int | 0 - 3000 | 0 | Degré non conventionnel/surréaliste |
| Expérimental | `--exp N` | int | 0 - 100 | 0 | Effet esthétique (exclusif V7) |
| Mode Raw | `--raw` | Drapeau | — | Désactivé | Désactiver l'embellissement par défaut |
| Mosaïque | `--tile` | Drapeau | — | Désactivé | Motif répétitif sans couture |
| Poids de l'image | `--iw N` | float | 0 - 3 | 1 | Influence du prompt image |
| Référence de style | `--sref [URL]` | URL | — | — | Correspondre au style visuel |
| Poids du style | `--sw N` | int | 0 - 1000 | 100 | Force de la référence de style |
| Version de style | `--sv N` | énumération | 1, 2, 3, 4, 5, 6 | 4 | Version de l'algorithme de style |
| Référence d'objet | `--oref [URL]` | URL | — | — | Placer un personnage/objet (exclusif V7), **coût doublé** |
| Poids de la référence d'objet | `--ow N` | int | 1 - 1000 | 100 | Force de la référence d'objet |
| Mode brouillon | `--draft` | drapeau | — | désactivé | 10x plus rapide, coût réduit de moitié (exclusif V7), contrôlé via API `model_params.speed` |

<Note>
Le mode de vitesse (draft / fast / turbo) est contrôlé via le champ API `model_params.speed`. Il n'est pas nécessaire d'écrire manuellement `--draft` / `--fast` / `--turbo` dans le prompt.
</Note>

## Paramètres de base

### Rapport d'aspect `--ar`

Définit le rapport d'aspect de l'image. Seuls les ratios entiers sont pris en charge.

```
un chat --ar 16:9
```

Valeurs courantes : `1:1`, `4:3`, `3:2`, `16:9`, `9:16`, `2:3`

### Chaos `--chaos` / `--c`

Contrôle la diversité des résultats générés. Des valeurs plus élevées produisent de plus grandes différences entre les 4 images.

```
un chat --c 50
```

| Plage | Effet |
|-------|-------|
| 0 | Les 4 images sont très similaires (par défaut) |
| 1-50 | Diversité modérée |
| 50-100 | Hautement aléatoire, adapté à l'exploration créative |

### Prompt négatif `--no`

Exclut les éléments indésirables, équivalent à un poids de -0,5. Prend en charge plusieurs mots-clés.

```
champ de tulipes vibrant --no rouge,ombres
```

<Warning>
V7 ne prend pas en charge la syntaxe multi-prompt `::`. Vous ne pouvez utiliser que `--no` pour exclure les éléments indésirables.
</Warning>

### Qualité `--quality` / `--q`

Contrôle le niveau de détail de la génération.

```
un chat --q 2
```

| Valeur | Description |
|--------|-------------|
| 1 | Par défaut, avec cohérence des mains optimisée |
| 2 | Utilise le modèle V7 avant optimisation |
| 4 | Mode expérimental (incompatible avec `--oref`) |

<Info>
Des valeurs de qualité plus élevées consomment davantage de ressources GPU et augmentent les coûts en conséquence.
</Info>

### Seed `--seed`

Fixe l'état aléatoire initial pour les tests comparatifs.

```
un chat --seed 23453422
```

- Plage : 0 - 4294967295
- Même seed + même prompt → résultats similaires (non identiques)
- Non applicable en mode de vitesse turbo

### Stylisation `--stylize` / `--s`

Ajuste l'équilibre entre réalisme et style artistique.

```
un chat --s 500
```

| Plage | Effet |
|-------|-------|
| 0-250 | Style réaliste, fidèle au prompt |
| 250-750 | Équilibré |
| 750-1000 | Expression artistique forte, couleurs et composition audacieuses |

### Weird `--weird` / `--w`

Introduit des éléments non conventionnels et surréalistes.

```
paysage spatial --w 800
```

| Plage | Effet |
|-------|-------|
| 10-300 | Légèrement étrange |
| 300-1000 | Éléments nettement non conventionnels |
| 1000-3000 | Surréalisme et abstraction extrêmes |

### Expérimental `--exp` (exclusif V7)

Similaire à `--stylize` mais cumulable, génère des images plus détaillées, dynamiques et créatives.

```
un chat --exp 25
```

- Valeurs recommandées : 5, 10, 25, 50, 100
- Changements d'effet notables de 5 à 50, rendements décroissants de 50 à 100
- Peut remplacer les effets de `--stylize` au-dessus de 25-50

### Mode Raw `--raw`

Désactive l'embellissement par défaut pour une adhérence plus stricte aux détails du prompt. Adapté au photoréalisme ou aux scénarios de contrôle précis.

```
photo de produit --raw
```

### Mosaïque `--tile`

Génère des motifs répétables sans couture (dans les quatre directions), adaptés aux textures, papiers peints et designs de tissus.

```
texture de mur de briques --tile
```

## Paramètres de référence d'image

### Prompt image

Placez les URLs d'images au **début** du prompt pour influencer le contenu généré.

```
https://example.com/ref.jpg description textuelle --iw 1.5
```

**Règles de combinaison valides :**

| Combinaison | Valide ? |
|-------------|----------|
| 1 image + pas de texte | ❌ Invalide (provoquera une erreur) |
| 1 image + description textuelle | ✅ Valide |
| 2+ images + pas de texte | ✅ Valide |
| 2+ images + description textuelle | ✅ Valide |

- Maximum 20 prompts image
- Formats pris en charge : `.png`, `.gif`, `.webp`, `.jpg`, `.jpeg`

### Poids de l'image `--iw`

Contrôle l'influence du prompt image sur le résultat. Plage 0 - 3, par défaut 1. Des valeurs plus élevées produisent des résultats plus proches de l'image de référence.

```
https://example.com/ref.jpg paysage aquarelle --iw 2.0
```

### Référence de style `--sref`

Correspond au style visuel (couleurs, textures, éclairage) de l'image de référence sans copier le contenu. **Doit être utilisé avec un prompt textuel**.

```
un chat --sref https://example.com/style.jpg
```

- Plusieurs prises en charge : `--sref URL1 URL2`
- Attribution de poids prise en charge : `--sref URL1::2 URL2::1` (Note : le `::` ici est la syntaxe de poids au sein du paramètre sref, un concept différent du séparateur multi-prompt `::` que V7 ne prend pas en charge)
- Style aléatoire pris en charge : `--sref random` (renvoie un code numérique après génération, réutilisable)
- Maximum 20 srefs

### Poids du style `--sw`

Contrôle l'influence de la référence de style. Plage 0 - 1000, par défaut 100.

```
un chat --sref https://example.com/style.jpg --sw 500
```

### Version de style `--sv`

Change la version de l'algorithme de référence de style.

| Version | Caractéristiques |
|---------|-----------------|
| 1 | Algorithme original, plus « atmosphérique » |
| 2 | Deuxième itération |
| 3 | Version mise à jour de v1 |
| 4 | Version mise à jour de v2 (**par défaut**) |
| 5 | Nouveau dans V7 |
| 6 | Nouveau dans V7 |

### Référence d'objet `--oref` (exclusif V7)

Place des personnages, objets, véhicules ou créatures non humaines de l'image de référence dans une nouvelle image. **Doit être utilisé avec un prompt textuel**.

```
un caneton tenant une baguette magique --oref https://example.com/duck.jpg
```

<Warning>
L'utilisation de la référence d'objet consomme 2x le temps GPU, doublant le coût. Une seule image de référence peut être utilisée.
</Warning>

### Poids de la référence d'objet `--ow` (exclusif V7)

Plage 1 - 1000, par défaut 100. Il est recommandé de rester en dessous de 400, sinon les résultats peuvent être imprévisibles. Des valeurs élevées de `--stylize` peuvent nécessiter un `--ow` plus élevé pour compenser.

```
caneton --oref https://example.com/duck.jpg --ow 300
```

## Modes de vitesse

Le mode de vitesse est contrôlé via le champ API `model_params.speed`, qui est automatiquement injecté dans le prompt.

| Mode | Description | Coût |
|------|-------------|------|
| `draft` | 10x plus rapide, qualité inférieure, peut être amélioré ultérieurement avec [mj-v7-enhance](/fr/api-manual/image-series/midjourney/mj-v7-enhance). **Uniquement pour le point de terminaison de génération d'images** | Le plus bas |
| `fast` | Qualité standard (par défaut) | Standard |
| `turbo` | Vitesse maximale | ~2x fast |

## Dépendances

Les paramètres suivants nécessitent d'autres paramètres pour prendre effet :

| Paramètre | Prérequis |
|-----------|-----------|
| `--sv`, `--sw` | Nécessite `--sref` |
| `--iw` | Nécessite un prompt image |
| `--ow` | Nécessite `--oref` |
| `--sref`, `--oref` | Nécessite un prompt textuel |
| Prompt image unique | Nécessite un prompt textuel |

---

## Conflits

| Paramètre A | Paramètre B | Description |
|-------------|-------------|-------------|
| `--oref` | Vitesse draft / turbo | Référence d'objet uniquement compatible avec le mode fast |
| `--oref` | `--q 4` | Qualité 4 incompatible avec la référence d'objet |
| `--oref` | inpaint/pan/outpaint | Les images de référence d'objet ne peuvent pas être utilisées pour ces opérations |
| `--draft` | `--tile` | Le mode draft ne prend pas en charge la mosaïque |
| Prompt image uniquement | `--stylize` / `--weird` | Incompatible sans texte |

<Tip>
Lorsque `--exp` dépasse 25-50, il supprime l'effet de `--stylize`. Il est recommandé d'utiliser des valeurs exp plus basses (5-25) lors de leur combinaison.
</Tip>

## Paramètres non pris en charge dans V7

| Paramètre | Description |
|-----------|-------------|
| `--stop` | Non pris en charge dans V7 |
| `--cref` / `--cw` | Utilisez plutôt `--oref` référence d'objet |
| `--relax` | V7 ne prend pas en charge le mode relax |
| `--repeat` / `--r` | Non pris en charge via API |
| `--p` (personalize) | Non pris en charge via API |
| `{}` permutations | Non pris en charge via API |
| `--stealth` / `--public` | Non pris en charge par l'API (les images ne sont pas accessibles publiquement) |
| Multi-prompt `::` | Non pris en charge dans V7 |

## Directives de format de prompt

### Structure de base

```
[URL d'image] description textuelle --param1 valeur1 --param2 valeur2
```

### Règles d'écriture

- Les paramètres se placent à la **fin** du prompt textuel
- Un **espace** est requis avant `--`
- Ne pas utiliser de ponctuation dans les paramètres
- Aucun texte ne peut suivre après les paramètres

### Exemples

<CodeGroup>
```text Correct
un chat persan jaune --ar 1:2 --s 500
```

```text Incorrect
un chat persan jaune--ar 2:3          ← espace manquant avant --
un chat persan jaune - - ar 2:3       ← espace entre --
un chat persan jaune --ar 2:3, --s 50 ← ponctuation dans les paramètres
un chat persan jaune --ar 2:3 plus de description ← texte après les paramètres
```
</CodeGroup>
