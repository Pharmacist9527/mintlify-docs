## Aperçu

OpenClaw Manager est un outil de gestion visuel qui vous aide à installer, configurer et gérer rapidement des instances OpenClaw. Aucune modification manuelle de configuration n'est nécessaire — seulement 3 étapes pour déployer.

Deux canaux de messagerie sont pris en charge :

| | Telegram | Feishu (Lark) |
|---|---|---|
| Connexion | Bot API (long polling) | Connexion persistante WebSocket |
| Identifiants requis | Bot Token + votre User ID | App ID + App Secret |
| Configuration de la plateforme | Créer un Bot uniquement | Créer une application + activer l'abonnement WebSocket |
| Authentification utilisateur | Lié par User ID — seuls les utilisateurs autorisés peuvent discuter | Tout utilisateur pouvant envoyer un message au Bot peut discuter |
| Idéal pour | Usage personnel, utilisateurs internationaux | Collaboration en équipe, utilisateurs en Chine |

## Vérification de l'Environnement Système

Avant l'installation, il est recommandé d'exécuter l'outil de vérification de l'environnement pour vous assurer que votre système répond aux exigences d'OpenClaw.

### Télécharger l'Outil de Vérification

Téléchargez l'outil de vérification pour votre plateforme depuis [GitHub Releases](https://github.com/suuuuuu-1/openclaw-env-checker/releases) :

| Plateforme | Nom du fichier |
|------|--------|
| Windows | `openclaw-checker-win-x64.exe` |
| macOS (Intel) | `openclaw-checker-macos-x64` |
| macOS (Apple Silicon) | `openclaw-checker-macos-arm64` |
| Linux | `openclaw-checker-linux-x64` |

### Éléments de Vérification

L'outil vérifie automatiquement les éléments suivants :

- ✅ Version de Node.js (>= 22.12.0 requis)
- ✅ npm disponible
- ✅ Git disponible
- ✅ Connectivité réseau (github.com, npmjs.org, evolink.ai)

![Exemple de vérification réussie](/images/OpenClaw-Checker/check-success.png)

Si la vérification échoue, l'outil fournira des suggestions de correction spécifiques.

## Prérequis

### 1. Installer Node.js

OpenClaw Manager nécessite Node.js 22 ou supérieur.

<Tabs>
  <Tab title="Windows">
    Visitez le [site officiel de Node.js](https://nodejs.org/fr/download), téléchargez l'installateur Windows (fichier .msi) et exécutez le programme d'installation.

    Après l'installation, ouvrez PowerShell pour vérifier :

    ```bash
    node --version
    npm --version
    ```

    <Note>
    Il est recommandé d'exécuter PowerShell en tant qu'administrateur pour éviter les problèmes de permissions lors de l'installation.
    </Note>
  </Tab>
  <Tab title="macOS">
    **Méthode 1 : Utiliser l'installateur**

    Visitez le [site officiel de Node.js](https://nodejs.org/fr/download), téléchargez l'installateur macOS (fichier .pkg) et exécutez le programme d'installation.

    **Méthode 2 : Utiliser Homebrew**

    ```bash
    brew install node
    ```

    Après l'installation, ouvrez le Terminal pour vérifier :

    ```bash
    node --version
    npm --version
    ```

    <Note>
    Si vous rencontrez des problèmes de permissions lors de l'installation, vous devrez peut-être ajouter `sudo` avant la commande.
    </Note>
  </Tab>
</Tabs>

### 2. Installer Git

OpenClaw dépend de Git pour la gestion des versions et l'installation des plugins.

<Tabs>
  <Tab title="Windows">
    Visitez le [site officiel de Git](https://git-scm.com/downloads), téléchargez l'installateur Windows, exécutez le programme d'installation et terminez l'installation avec les options par défaut.

    Après l'installation, ouvrez PowerShell pour vérifier :

    ```bash
    git --version
    ```
  </Tab>
  <Tab title="macOS">
    **Méthode 1 : Utiliser Homebrew (recommandé)**

    ```bash
    brew install git
    ```

    **Méthode 2 : Utiliser Xcode Command Line Tools**

    ```bash
    xcode-select --install
    ```

    Après l'installation, ouvrez le Terminal pour vérifier :

    ```bash
    git --version
    ```
  </Tab>
</Tabs>

### 3. Obtenir une clé API EvoLink

- Connectez-vous à la [console EvoLink](https://evolink.ai/dashboard)
- Trouvez les clés API dans la console, cliquez sur « Créer une nouvelle clé », puis copiez la clé générée
- La clé API commence généralement par `sk-`, veuillez la conserver en sécurité

### 4. Identifiants du canal

<Accordion title="Guide de configuration des identifiants Telegram / Feishu">
<Tabs>
  <Tab title="Telegram">

Les étapes restantes — abonnement aux événements et publication de la version — doivent être effectuées après le déploiement et le démarrage de la passerelle, sinon les paramètres de connexion longue ne pourront pas être sauvegardés. ## Étape 1 : Télécharger et exécuter

Téléchargez l'exécutable correspondant à votre plateforme depuis [Releases](https://github.com/Pharmacist9527/openclaw-manager/releases) :

![Page des releases GitHub](/images/OpenClaw-Auto/github-releases.png)

![Télécharger l'exécutable](/images/OpenClaw-Auto/github-repo.png)

| Plateforme | Fichier |
|----------|------|
| Windows | `openclaw-manager-win-x64.exe` |
| macOS (Intel) | `openclaw-manager-macos-x64` |
| macOS (Apple Silicon) | `openclaw-manager-macos-arm64` |
| Linux | `openclaw-manager-linux-x64` |

<Warning>
Assurez-vous que [Node.js 22+](https://nodejs.org/fr/download) est installé.
</Warning>

Les utilisateurs Windows font un clic droit et exécutent en tant qu'administrateur. Les utilisateurs macOS / Linux exécutent avec `sudo` dans le terminal :

```bash
chmod +x ./openclaw-manager && sudo ./openclaw-manager
```

Le script vérifiera et installera OpenClaw si nécessaire, puis démarrera l'interface de gestion web locale et ouvrira votre navigateur.

![Créer une nouvelle instance](/images/OpenClaw-Auto/home-new-instance.png)

## Étape 2 : Créer une instance

### 2.1 — Informations de base

| Champ | Description |
|-------|-------------|
| Nom de l'instance | Nom de votre instance (ex. `mybot`), doit être unique sur cette machine |
| Modèle | Choisissez un modèle : Haiku 4.5 / Sonnet 4.5 / Opus 4.5 / Opus 4.6 |
| Canal | Choisissez un canal : Telegram ou Feishu |

![Informations de base](/images/OpenClaw-Auto/wizard-step1-basic-info.png)

### 2.2 — Identifiants et déploiement

Tous les canaux nécessitent une **clé API EvoLink**.

<Tabs>
  <Tab title="Telegram">
    Entrez le **Bot Token** obtenu de @BotFather, puis cliquez sur **Deploy**.

    Le script effectuera automatiquement :
    1. La création du fichier de configuration OpenClaw
    2. L'écriture des paramètres du modèle et de la clé API
    3. La configuration et l'activation du plugin Telegram
    4. L'installation et le démarrage du service de passerelle

    ![Identifiants Telegram](/images/OpenClaw-Auto/wizard-step2-telegram-credentials.png)
  </Tab>
  <Tab title="Feishu">
    Entrez l'**App ID** et l'**App Secret** depuis Feishu Open Platform, puis cliquez sur **Deploy**.

    Le script effectuera automatiquement :
    1. La création du fichier de configuration OpenClaw
    2. L'écriture des paramètres du modèle et de la clé API
    3. La configuration du plugin Feishu (écriture de l'App ID / App Secret) et son activation
    4. L'installation et le démarrage du service de passerelle

    ![Identifiants Feishu](/images/OpenClaw-Auto/wizard-step2-credentials.png)
  </Tab>
</Tabs>

Attendez que la barre de progression se termine, puis vous passerez automatiquement à l'étape suivante.

### 2.3 — Connexion

<Tabs>
  <Tab title="Telegram">
    Entrez votre **Telegram User ID** (l'identifiant numérique obtenu de @userinfobot), puis cliquez sur **Connect**.

    Le script écrit cet identifiant dans la configuration `allowedUsers` — seul cet utilisateur pourra discuter avec le Bot. Une fois connecté, vous serez redirigé vers la page d'accueil.

    ![Connexion Telegram](/images/OpenClaw-Auto/wizard-step3-telegram-connect.png)
  </Tab>
  <Tab title="Feishu">
    Le déploiement est terminé. Retournez maintenant sur Feishu Open Platform pour configurer l'abonnement aux événements.

    <Warning>
    Avant de configurer l'abonnement aux événements, assurez-vous que les éléments suivants sont effectués :
    - La configuration du canal Feishu est terminée (voir le déploiement de l'Étape 2)
    - La passerelle est en cours d'exécution (vérifiez avec `openclaw gateway status`)
    </Warning>

    Sur la page **Abonnement aux événements** :

    1. Sélectionnez **Utiliser la connexion longue pour recevoir les événements** (mode WebSocket)
    2. Ajoutez l'événement : `im.message.receive_v1` (recevoir des messages)

    <Warning>
    Si la passerelle n'est pas démarrée ou si le canal n'a pas été ajouté, les paramètres de connexion longue ne pourront pas être sauvegardés.
    </Warning>

    ![Abonnement aux événements](/images/OpenClaw-Feishu/event-subscription.avif)

    3. Sous « Gestion des versions et publication », créez une version et soumettez-la pour examen

    <Note>
    Les applications Feishu nécessitent l'approbation d'un administrateur avant de pouvoir être utilisées. Les applications d'entreprise personnalisées sont généralement approuvées automatiquement.
    </Note>

    Cliquez sur **Done** — le script redémarrera la passerelle pour établir la connexion WebSocket Feishu. Après la redirection vers la page d'accueil, vous pouvez commencer à envoyer des messages au Bot sur Feishu.

    ![Connexion Feishu](/images/OpenClaw-Auto/wizard-step3-feishu-connect.png)
  </Tab>
</Tabs>

## Gérer les instances

Après le déploiement, la page d'accueil affiche toutes les instances :

![Gestion des instances](/images/OpenClaw-Auto/home-instance-running.png)

| Statut | Description |
|--------|-------------|
| 🟢 Vert | Passerelle en cours d'exécution |
| 🔴 Rouge | Passerelle arrêtée |

Chaque carte d'instance affiche le modèle et le canal utilisés. Actions disponibles :

- **Démarrer / Arrêter** : Démarrer ou arrêter le service de passerelle
- **Supprimer** : Supprimer l'instance (arrête la passerelle et nettoie tous les fichiers de configuration)

Cliquez sur **+ New Instance** pour créer plusieurs instances, chacune fonctionnant indépendamment sur un port différent.
