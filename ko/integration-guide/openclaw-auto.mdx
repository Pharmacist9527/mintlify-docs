## 개요

OpenClaw Manager는 OpenClaw 인스턴스를 빠르게 설치, 구성 및 관리할 수 있도록 도와주는 시각적 관리 도구입니다. 수동으로 설정 파일을 편집할 필요 없이 3단계만으로 배포할 수 있습니다.

두 가지 메시징 채널을 지원합니다:

| | Telegram | Feishu (Lark) |
|---|---|---|
| 연결 방식 | Bot API (롱 폴링) | WebSocket 영구 연결 |
| 필요한 자격 증명 | Bot Token + 사용자 ID | App ID + App Secret |
| 플랫폼 설정 | Bot만 생성 | 앱 생성 + WebSocket 구독 활성화 |
| 사용자 인증 | 사용자 ID로 바인딩 — 인증된 사용자만 채팅 가능 | Bot에 메시지를 보낼 수 있는 모든 사용자가 채팅 가능 |
| 적합한 용도 | 개인 사용, 해외 사용자 | 팀 협업, 중국 내 사용자 |

## 사전 준비

### 공통

- **Node.js 22+**: 스크립트가 npm을 통해 OpenClaw를 설치하므로 Node.js가 필요합니다
- **EvoLink API Key**: [evolink.ai](https://evolink.ai/dashboard)에서 가입하고 API Key를 발급받으세요

## Step 1: 다운로드 및 실행

[Releases](https://github.com/open-claw/openclaw-manager/releases)에서 사용 중인 플랫폼에 맞는 실행 파일을 다운로드하세요:

![GitHub Releases page](/images/OpenClaw-Auto/github-releases.png)

![Download executable](/images/OpenClaw-Auto/github-repo.png)

| 플랫폼 | 파일 |
|----------|------|
| Windows | `openclaw-manager-win-x64.exe` |
| macOS (Intel) | `openclaw-manager-macos-x64` |
| macOS (Apple Silicon) | `openclaw-manager-macos-arm64` |
| Linux | `openclaw-manager-linux-x64` |

<Warning>
[Node.js 22+](https://nodejs.org/ko/download)가 설치되어 있는지 확인하세요.
</Warning>

Windows 사용자는 마우스 오른쪽 버튼을 클릭하여 관리자 권한으로 실행하세요. macOS / Linux 사용자는 터미널에서 `sudo`로 실행하세요:

```bash
chmod +x ./openclaw-manager && sudo ./openclaw-manager
```

스크립트가 OpenClaw를 확인하고 필요 시 설치한 후, 로컬 웹 관리 인터페이스를 시작하고 브라우저를 엽니다.

![Create new instance](/images/OpenClaw-Auto/home-new-instance.png)

## Step 2: 인스턴스 생성

### 2.1 — 기본 정보

| 필드 | 설명 |
|-------|-------------|
| 인스턴스 이름 | 인스턴스의 이름 (예: `mybot`), 이 머신에서 고유해야 합니다 |
| 모델 | 모델 선택: Haiku 4.5 / Sonnet 4.5 / Opus 4.5 / Opus 4.6 |
| 채널 | 채널 선택: Telegram 또는 Feishu |

![Basic info](/images/OpenClaw-Auto/wizard-step1-basic-info.png)

### 2.2 — 자격 증명 및 배포

모든 채널에 **EvoLink API Key**가 필요합니다.

<Tabs>
  <Tab title="Telegram">
    @BotFather에서 받은 **Bot Token**을 입력한 후 **Deploy**를 클릭하세요.

    스크립트가 자동으로 다음을 수행합니다:
    1. OpenClaw 설정 파일 생성
    2. 모델 및 API Key 설정 작성
    3. Telegram 플러그인 구성 및 활성화
    4. 게이트웨이 서비스 설치 및 시작

    ![Telegram credentials](/images/OpenClaw-Auto/wizard-step2-telegram-credentials.png)
  </Tab>
  <Tab title="Feishu">
    Feishu Open Platform에서 받은 **App ID**와 **App Secret**을 입력한 후 **Deploy**를 클릭하세요.

    스크립트가 자동으로 다음을 수행합니다:
    1. OpenClaw 설정 파일 생성
    2. 모델 및 API Key 설정 작성
    3. Feishu 플러그인 구성 (App ID / App Secret 작성) 및 활성화
    4. 게이트웨이 서비스 설치 및 시작

    ![Feishu credentials](/images/OpenClaw-Auto/wizard-step2-credentials.png)
  </Tab>
</Tabs>

진행 바가 완료될 때까지 기다리면 자동으로 다음 단계로 이동합니다.

### 2.3 — 연결

<Tabs>
  <Tab title="Telegram">
    **Telegram 사용자 ID** (@userinfobot에서 받은 숫자 ID)를 입력한 후 **Connect**를 클릭하세요.

    스크립트가 이 ID를 `allowedUsers` 설정에 작성합니다 — 이 사용자만 Bot과 채팅할 수 있습니다. 연결이 완료되면 홈 페이지로 리디렉션됩니다.

    ![Telegram connect](/images/OpenClaw-Auto/wizard-step3-telegram-connect.png)
  </Tab>
  <Tab title="Feishu">
    배포가 완료되었습니다. 이제 Feishu Open Platform으로 돌아가서 이벤트 구독을 설정하세요.

    <Warning>
    이벤트 구독을 설정하기 전에 다음 사항이 완료되었는지 확인하세요:
    - Feishu 채널 구성이 완료됨 (Step 2 배포 참조)
    - 게이트웨이가 실행 중임 (`openclaw gateway status`로 확인)
    </Warning>

    **이벤트 구독** 페이지에서:

    1. **롱 커넥션을 사용하여 이벤트 수신**을 선택하세요 (WebSocket 모드)
    2. 이벤트 추가: `im.message.receive_v1` (메시지 수신)

    <Warning>
    게이트웨이가 시작되지 않았거나 채널이 추가되지 않은 경우 롱 커넥션 설정 저장에 실패합니다.
    </Warning>

    ![Event subscription](/images/OpenClaw-Feishu/event-subscription.avif)

    3. "버전 관리 및 출시"에서 버전을 생성하고 검토를 위해 제출하세요

    <Note>
    Feishu 앱은 사용하기 전에 관리자 승인이 필요합니다. 기업 사용자 정의 앱은 일반적으로 자동 승인됩니다.
    </Note>

    **Done**을 클릭하세요 — 스크립트가 게이트웨이를 재시작하여 Feishu WebSocket 연결을 설정합니다. 홈 페이지로 리디렉션된 후 Feishu에서 Bot에 메시지를 보낼 수 있습니다.

    ![Feishu connect](/images/OpenClaw-Auto/wizard-step3-feishu-connect.png)
  </Tab>
</Tabs>

## 인스턴스 관리

배포 후 홈 페이지에 모든 인스턴스가 표시됩니다:

![Instance management](/images/OpenClaw-Auto/home-instance-running.png)

| 상태 | 설명 |
|--------|-------------|
| 🟢 초록색 | 게이트웨이 실행 중 |
| 🔴 빨간색 | 게이트웨이 중지됨 |

각 인스턴스 카드에는 사용 중인 모델과 채널이 표시됩니다. 사용 가능한 작업:

- **Start / Stop**: 게이트웨이 서비스 시작 또는 중지
- **Delete**: 인스턴스 삭제 (게이트웨이를 중지하고 모든 설정 파일을 정리)

**+ New Instance**를 클릭하여 여러 인스턴스를 생성할 수 있으며, 각 인스턴스는 서로 다른 포트에서 독립적으로 실행됩니다.
