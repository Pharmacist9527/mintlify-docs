## 매개변수 개요

| 매개변수 | 작성법 | 유형 | 값 범위 | 기본값 | 설명 |
|------|------|------|---------|--------|------|
| 화면 비율 | `--ar W:H` | 정수 비율 | 임의의 양의 정수 비율 | 1:1 | 이미지 화면 비율 |
| 카오스 | `--c N` | int | 0 - 100 | 0 | 생성 결과의 다양성 |
| 부정 프롬프트 | `--no 단어1,단어2` | 키워드 | — | — | 제외할 요소 지정 |
| 품질 | `--q N` | 열거형 | 1, 2, 4 | 1 | 생성 정밀도, **비용에 영향** |
| 시드 | `--seed N` | int | 0 - 4294967295 | 무작위 | 시드를 고정하여 결과 재현 |
| 스타일라이즈 | `--s N` | int | 0 - 1000 | 100 | 예술적 스타일 강도 |
| 위어드 | `--w N` | int | 0 - 3000 | 0 | 비일상적/초현실적 정도 |
| 실험 매개변수 | `--exp N` | int | 0 - 100 | 0 | 미적 효과 (V7 전용) |
| RAW 모드 | `--raw` | 플래그 | — | 꺼짐 | 기본 미화 비활성화 |
| 타일 | `--tile` | 플래그 | — | 꺼짐 | 심리스 반복 패턴 |
| 이미지 가중치 | `--iw N` | float | 0 - 3 | 1 | 참조 이미지 영향도 |
| 스타일 참조 | `--sref [URL]` | URL | — | — | 비주얼 스타일 매칭 |
| 스타일 가중치 | `--sw N` | int | 0 - 1000 | 100 | 스타일 참조 강도 |
| 스타일 버전 | `--sv N` | 열거 | 1, 2, 3, 4, 5, 6 | 4 | 스타일 알고리즘 버전 |
| 오브젝트 참조 | `--oref [URL]` | URL | — | — | 캐릭터/오브젝트 배치 (V7 전용), **비용 2배** |
| 오브젝트 참조 가중치 | `--ow N` | int | 1 - 1000 | 100 | 오브젝트 참조 강도 |
| 드래프트 모드 | `--draft` | 플래그 | — | 꺼짐 | 10배 빠름, 비용 절반 (V7 전용), API `model_params.speed`로 제어 |

<Note>
속도 모드(draft / fast / turbo)는 API의 `model_params.speed` 필드로 제어합니다. 프롬프트에 `--draft` / `--fast` / `--turbo`를 직접 작성할 필요가 없습니다.
</Note>

## 기본 매개변수

### 화면 비율 `--ar`

이미지의 화면 비율을 설정합니다. 정수 비율만 지원됩니다.

```
고양이 --ar 16:9
```

일반적인 값: `1:1`, `4:3`, `3:2`, `16:9`, `9:16`, `2:3`

### 카오스 `--chaos` / `--c`

생성 결과의 다양성을 제어합니다. 값이 높을수록 4장의 이미지 간 차이가 커집니다.

```
고양이 --c 50
```

| 범위 | 효과 |
|------|------|
| 0 | 4장의 이미지가 매우 유사 (기본값) |
| 1-50 | 적당한 다양성 |
| 50-100 | 높은 무작위성, 창의적 탐색에 적합 |

### 부정 프롬프트 `--no`

제외할 요소를 지정합니다. -0.5 가중치와 동일합니다. 여러 키워드를 지원합니다.

```
활기찬 튤립 밭 --no 빨강,그림자
```

<Warning>
V7은 `::` 다중 프롬프트 구문을 지원하지 않습니다. 원하지 않는 요소를 제외하려면 `--no`만 사용할 수 있습니다.
</Warning>

### 품질 `--quality` / `--q`

생성의 정밀도를 제어합니다.

```
고양이 --q 2
```

| 값 | 설명 |
|----|------|
| 1 | 기본값, 손의 일관성 최적화 |
| 2 | 최적화 전 V7 모델 사용 |
| 4 | 실험 모드 (`--oref`와 호환되지 않음) |

<Info>
품질 값이 높을수록 GPU 소비가 증가하고 비용도 그에 따라 증가합니다.
</Info>

### 시드 `--seed`

초기 무작위 상태를 고정하여 비교 테스트에 사용합니다.

```
고양이 --seed 23453422
```

- 범위: 0 - 4294967295
- 동일 시드 + 동일 프롬프트 → 유사한 (완전히 동일하지는 않은) 결과
- turbo 속도 모드에는 적용 불가

### 스타일라이즈 `--stylize` / `--s`

사실적 표현과 예술적 표현 사이의 균형을 조절합니다.

```
고양이 --s 500
```

| 범위 | 효과 |
|------|------|
| 0-250 | 사실적 스타일, 프롬프트에 충실 |
| 250-750 | 균형 |
| 750-1000 | 강한 예술성, 대담한 색상과 구도 |

### 위어드 `--weird` / `--w`

비일상적이고 초현실적인 요소를 도입합니다.

```
우주 풍경 --w 800
```

| 범위 | 효과 |
|------|------|
| 10-300 | 약간의 위어드 |
| 300-1000 | 뚜렷한 비전통적 요소 |
| 1000-3000 | 극단적 초현실주의와 추상 |

### 실험 매개변수 `--exp` (V7 전용)

`--stylize`와 유사하지만 중첩 사용이 가능하며, 더 상세하고 역동적이며 창의적인 이미지를 생성합니다.

```
고양이 --exp 25
```

- 권장 값: 5, 10, 25, 50, 100
- 5-50에서 효과 변화가 뚜렷하고, 50-100에서는 변화가 적음
- 25-50을 초과하면 `--stylize` 효과를 덮어쓸 수 있음

### RAW 모드 `--raw`

기본 미화를 비활성화하고 프롬프트 세부사항을 더 엄격하게 따릅니다. 사실감 또는 정밀한 제어를 추구하는 장면에 적합합니다.

```
제품 사진 --raw
```

### 타일 `--tile`

심리스 반복 가능한 패턴(상하좌우 4방향 심리스 연결)을 생성합니다. 텍스처, 배경화면, 직물 디자인에 적합합니다.

```
벽돌 벽 텍스처 --tile
```

## 이미지 참조 매개변수

### 이미지 프롬프트 (참조 이미지)

프롬프트 **앞부분**에 이미지 URL을 배치하여 이미지로 생성 내용에 영향을 줍니다.

```
https://example.com/ref.jpg 텍스트 설명 --iw 1.5
```

**유효한 조합 규칙:**

| 조합 방식 | 유효 여부 |
|---------|---------|
| 이미지 1장 + 텍스트 없음 | ❌ 무효 (오류 발생) |
| 이미지 1장 + 텍스트 설명 | ✅ 유효 |
| 이미지 2장 이상 + 텍스트 없음 | ✅ 유효 |
| 이미지 2장 이상 + 텍스트 설명 | ✅ 유효 |

- 최대 20장의 참조 이미지
- 지원 형식: `.png`, `.gif`, `.webp`, `.jpg`, `.jpeg`

### 이미지 가중치 `--iw`

참조 이미지가 결과에 미치는 영향도를 제어합니다. 범위 0 - 3, 기본값 1, 값이 높을수록 참조 이미지에 가까워집니다.

```
https://example.com/ref.jpg 수채화 풍경 --iw 2.0
```

### 스타일 참조 `--sref`

참조 이미지의 비주얼 스타일(색상, 텍스처, 조명)에 매칭합니다. 내용은 복사하지 않습니다. **텍스트 프롬프트와 함께 사용해야 합니다**.

```
고양이 --sref https://example.com/style.jpg
```

- 다중 지원: `--sref URL1 URL2`
- 가중치 할당 지원: `--sref URL1::2 URL2::1` (참고: 여기서 `::`는 sref 매개변수 내의 가중치 구문으로, V7에서 지원하지 않는 다중 프롬프트 `::` 구분자와는 다른 개념입니다)
- 랜덤 스타일 지원: `--sref random` (생성 후 숫자 코드가 반환되며 재사용 가능)
- 최대 20개 sref

### 스타일 가중치 `--sw`

스타일 참조의 영향 정도를 제어합니다. 범위 0 - 1000, 기본값 100.

```
고양이 --sref https://example.com/style.jpg --sw 500
```

### 스타일 버전 `--sv`

스타일 참조의 알고리즘 버전을 전환합니다.

| 버전 | 특징 |
|------|------|
| 1 | 오리지널 알고리즘, 더 "분위기 있는" 결과 |
| 2 | 두 번째 반복 |
| 3 | v1 업데이트 버전 |
| 4 | v2 업데이트 버전 (**기본값**) |
| 5 | V7에서 추가 |
| 6 | V7에서 추가 |

### 오브젝트 참조 `--oref` (V7 전용)

참조 이미지의 캐릭터, 오브젝트, 차량 또는 비인간 생물을 새 이미지에 배치합니다. **텍스트 프롬프트와 함께 사용해야 합니다**.

```
마법 지팡이를 든 작은 오리 --oref https://example.com/duck.jpg
```

<Warning>
오브젝트 참조를 사용하면 GPU 시간이 2배 소비되며 비용이 2배가 됩니다. 또한 참조 이미지는 1장만 사용할 수 있습니다.
</Warning>

### 오브젝트 참조 가중치 `--ow` (V7 전용)

범위 1 - 1000, 기본값 100. 400 이하로 유지하는 것을 권장하며, 그렇지 않으면 결과가 예측 불가능할 수 있습니다. 높은 `--stylize` 값은 보상을 위해 더 높은 `--ow`가 필요할 수 있습니다.

```
작은 오리 --oref https://example.com/duck.jpg --ow 300
```

## 속도 모드

속도 모드는 API의 `model_params.speed` 필드로 제어되며, 시스템이 자동으로 프롬프트에 주입합니다.

| 모드 | 설명 | 비용 |
|------|------|------|
| `draft` | 10배 빠름, 품질은 낮음, 이후 [mj-v7-enhance](/ko/api-manual/image-series/midjourney/mj-v7-enhance)로 향상 가능. **이미지 생성 인터페이스 전용** | 최저 |
| `fast` | 표준 품질 (기본값) | 표준 |
| `turbo` | 최고 속도 | fast의 약 2배 |

## 의존 관계

다음 매개변수는 다른 매개변수와 함께 사용해야 작동합니다:

| 매개변수 | 전제 의존 |
|------|---------|
| `--sv`, `--sw` | `--sref` 필요 |
| `--iw` | 이미지 프롬프트 (참조 이미지) 필요 |
| `--ow` | `--oref` 필요 |
| `--sref`, `--oref` | 텍스트 프롬프트 필요 |
| 단일 참조 이미지 | 텍스트 프롬프트 필요 |

---

## 충돌 관계

| 매개변수 A | 매개변수 B | 설명 |
|--------|--------|------|
| `--oref` | draft / turbo 속도 | 오브젝트 참조는 fast 모드만 호환 |
| `--oref` | `--q 4` | 품질 4는 오브젝트 참조와 호환되지 않음 |
| `--oref` | inpaint/pan/outpaint | 오브젝트 참조 이미지는 이러한 작업에 사용 불가 |
| `--draft` | `--tile` | 드래프트 모드는 타일을 지원하지 않음 |
| 이미지 전용 프롬프트 | `--stylize` / `--weird` | 텍스트 없이는 호환되지 않음 |

<Tip>
`--exp` 값이 25-50을 초과하면 `--stylize` 효과를 억제합니다. 함께 사용할 때는 exp를 낮은 값(5-25)으로 설정하는 것을 권장합니다.
</Tip>

## V7에서 지원하지 않는 매개변수

| 매개변수 | 설명 |
|------|------|
| `--stop` | V7에서 미지원 |
| `--cref` / `--cw` | 대신 `--oref` 오브젝트 참조 사용 |
| `--relax` | V7에서 슬로우 모드 미지원 |
| `--repeat` / `--r` | API에서 미지원 |
| `--p` (personalize) | API에서 미지원 |
| `{}` 열거 | API에서 순열 프롬프트 미지원 |
| `--stealth` / `--public` | API에서 지원하지 않음 (이미지는 공개되지 않음) |
| 다중 프롬프트 `::` | V7에서 미지원 |

## 프롬프트 작성 규칙

### 기본 구조

```
[이미지URL] 텍스트 설명 --매개변수1 값1 --매개변수2 값2
```

### 작성 규칙

- 매개변수는 텍스트 프롬프트의 **끝**에 배치
- `--` 앞에는 반드시 **공백**이 필요
- 매개변수 안에 구두점을 사용하지 않음
- 매개변수 뒤에 텍스트를 작성하지 않음

### 예시

<CodeGroup>
```text 올바름
노란 페르시안 고양이 --ar 1:2 --s 500
```

```text 잘못됨
노란 페르시안 고양이--ar 2:3          ← -- 앞에 공백 없음
노란 페르시안 고양이 - - ar 2:3       ← -- 사이에 공백 있음
노란 페르시안 고양이 --ar 2:3, --s 50 ← 매개변수에 구두점 있음
노란 페르시안 고양이 --ar 2:3 추가 설명 ← 매개변수 뒤에 텍스트 있음
```
</CodeGroup>
