## 매개변수 개요

| 매개변수 | 작성법 | 유형 | 값 범위 | 기본값 | 설명 |
|------|------|------|---------|--------|------|
| 화면 비율 | `--ar W:H` | 정수 비율 | 임의의 양의 정수 비율 | 1:1 | 이미지 화면 비율 |
| 카오스 | `--c N` | int | 0 - 100 | 0 | 생성 결과의 다양성 |
| 시드 | `--seed N` | int | 0 - 4294967295 | 무작위 | 시드를 고정하여 결과 재현 |
| 스타일라이즈 | `--s N` | int | 0 - 1000 | 100 | 예술적 스타일 강도 |
| 실험 매개변수 | `--exp N` | int | 0 - 100 | 0 | 미적 효과, 스타일라이즈와 중첩 가능 |
| RAW 모드 | `--raw` | 플래그 | — | 꺼짐 | 기본 미화 비활성화 |
| 이미지 가중치 | `--iw N` | float | 0 - 3 | 1 | 참조 이미지 영향도 |
| 스타일 참조 | `--sref [URL]` | URL | — | — | 비주얼 스타일 매칭 |
| 스타일 가중치 | `--sw N` | int | 0 - 1000 | 100 | 스타일 참조 강도 |
| 오브젝트 참조 | `--oref [URL]` | URL | — | — | 캐릭터/오브젝트를 화면에 배치 |
| 오브젝트 참조 가중치 | `--ow N` | int | 1 - 1000 | 100 | 오브젝트 참조 강도 |

<Note>
두 가지 설정은 프롬프트에 작성하지 **않고**, API를 통해 제어합니다(프롬프트에 작성해도 적용되지 않습니다):

- **속도**(`draft` / `fast`) → `model_params.speed`
- **출력 품질**(`standard` / `hd`) → 최상위 `quality` 매개변수

`--v` / `--version`은 V8.1로 고정되며, `--niji`는 지원되지 않습니다. 아래 [속도 모드](#speed-modes)와 [출력 품질](#output-quality)을 참고하세요.
</Note>

## 기본 매개변수

### 화면 비율 `--ar`

이미지의 화면 비율을 설정합니다. 정수 비율만 지원됩니다.

```
a cat --ar 16:9
```

일반적인 값: `1:1`, `4:3`, `3:2`, `16:9`, `9:16`, `2:3`. 극단적인 비율은 예측 불가능한 결과를 만들 수 있습니다.

### 카오스 `--chaos` / `--c`

생성 결과의 다양성을 제어합니다. 값이 높을수록 4장의 이미지 간 차이가 커집니다.

```
a cat --c 50
```

| 범위 | 효과 |
|------|------|
| 0 | 4장의 이미지가 매우 유사 (기본값) |
| 1-50 | 적당한 다양성 |
| 50-100 | 높은 무작위성, 창의적 탐색에 적합 |

### 스타일라이즈 `--stylize` / `--s`

사실적 표현과 예술적 표현 사이의 균형을 조절합니다.

```
a cat --s 500
```

| 범위 | 효과 |
|------|------|
| 0-250 | 사실적 스타일, 프롬프트에 충실 |
| 250-750 | 균형 |
| 750-1000 | 강한 예술성, 대담한 색상과 구도 |

### 실험 매개변수 `--exp`

`--stylize`와 유사하지만 중첩 사용이 가능하며, 더 상세하고 역동적이며 창의적인 이미지를 생성합니다.

```
a cat --exp 25
```

- 권장 값: 5, 10, 25, 50, 100
- 5-50에서 효과 변화가 뚜렷하고, 50-100에서는 변화가 적음
- 25-50을 초과하면 `--stylize` 효과를 덮어쓸 수 있음

### RAW 모드 `--raw`

기본 미화를 비활성화하고 프롬프트 세부사항을 더 엄격하게 따릅니다. 사실감 또는 정밀한 제어를 추구하는 장면에 적합합니다.

```
product photo --raw
```

### 시드 `--seed`

초기 무작위 상태를 고정하여 비교 테스트에 사용합니다.

```
a cat --seed 23453422
```

- 범위: 0 - 4294967295
- 동일 시드 + 동일 프롬프트 → 유사한 (완전히 동일하지는 않은) 결과

## 이미지 참조 매개변수

### 이미지 프롬프트 (참조 이미지)

프롬프트 **앞부분**에 이미지 URL을 배치하여 이미지로 생성 내용에 영향을 줍니다.

```
https://example.com/ref.jpg 텍스트 설명 --iw 1.5
```

**유효한 조합 규칙:**

| 조합 방식 | 유효 여부 |
|---------|---------|
| 이미지 1장 + 텍스트 없음 | ❌ 무효 (오류 발생) |
| 이미지 1장 + 텍스트 설명 | ✅ 유효 |
| 이미지 2장 이상 + 텍스트 없음 | ✅ 유효 |
| 이미지 2장 이상 + 텍스트 설명 | ✅ 유효 |

- 최대 20장의 참조 이미지
- 지원 형식: `.png`, `.gif`, `.webp`, `.jpg`, `.jpeg`
- `draft` 속도에서는 사용 불가 (`fast` 사용)

### 이미지 가중치 `--iw`

참조 이미지가 결과에 미치는 영향도를 제어합니다. 범위 0 - 3, 기본값 1, 값이 높을수록 참조 이미지에 가까워집니다.

```
https://example.com/ref.jpg watercolor landscape --iw 2.0
```

### 스타일 참조 `--sref`

참조 이미지의 비주얼 스타일(색상, 텍스처, 조명)에 매칭합니다. 내용은 복사하지 않습니다. **텍스트 프롬프트와 함께 사용해야 합니다**.

```
a cat --sref https://example.com/style.jpg
```

- 다중 지원: `--sref URL1 URL2`
- 랜덤 스타일 지원: `--sref random` (생성 후 숫자 코드가 반환되며 재사용 가능)
- 최대 20개 sref

### 스타일 가중치 `--sw`

스타일 참조의 영향 정도를 제어합니다. 범위 0 - 1000, 기본값 100.

```
a cat --sref https://example.com/style.jpg --sw 500
```

### 오브젝트 참조 `--oref`

참조 이미지의 캐릭터, 오브젝트, 차량 또는 비인간 생물을 새 이미지에 배치합니다. **텍스트 프롬프트와 함께 사용해야 합니다**.

```
a duckling holding a magic wand --oref https://example.com/duck.jpg
```

<Warning>
오브젝트 참조 이미지는 1장만 사용할 수 있으며, 활성화하면 생성 시간이 더 길어집니다.
</Warning>

### 오브젝트 참조 가중치 `--ow`

범위 1 - 1000, 기본값 100. 400 이하로 유지하는 것을 권장하며, 그렇지 않으면 결과가 예측 불가능할 수 있습니다. 높은 `--stylize` 값은 보상을 위해 더 높은 `--ow`가 필요할 수 있습니다.

```
duckling --oref https://example.com/duck.jpg --ow 300
```

## 속도 모드

속도 모드는 API의 `model_params.speed` 필드로 제어합니다(프롬프트에 `--draft` / `--fast`를 작성하지 마세요).

| 모드 | 설명 | 비용 |
|------|------|------|
| `draft` | 단일 실행으로 24장의 경량 0.5K 스케치 이미지를 반환합니다. 이미지 생성 인터페이스 전용. | fast와 동일 배율 |
| `fast` | 표준 모드 (기본값), 매회 4장 생성 | 표준 |

<Note>
V7과 달리 V8.1의 `draft` 모드는 비용이 절반이 **아니며**, `fast`와 동일한 배율을 사용합니다. 드래프트 모드는 단일 실행으로 24장의 작은 스케치를 반환하므로, 마음에 드는 것을 골라 완전한 품질로 다시 실행하세요. 드래프트 모드는 `hd` 품질과 함께 사용할 수 없습니다.
</Note>

## 출력 품질

출력 해상도는 프롬프트 플래그가 **아닌** 최상위 `quality` 매개변수로 제어합니다. 프롬프트에 `--hd`를 작성하지 마세요.

| 값 | 해상도 | 비용 |
|----|--------|------|
| `standard` | 표준 해상도 (기본값) | 1배 |
| `hd` | 네이티브 2K HD 출력 | 1.5배 |

- 품질 배율은 속도 배율과 **상승(곱셈)** 관계입니다.
- `hd`는 `draft` 속도와 **호환되지 않습니다**.

## 의존 관계

다음 매개변수는 다른 매개변수와 함께 사용해야 작동합니다:

| 매개변수 | 전제 의존 |
|------|---------|
| `--sw` | `--sref` 필요 |
| `--ow` | `--oref` 필요 |
| `--iw` | 이미지 프롬프트 (참조 이미지) 필요 |
| `--sref`, `--oref` | 텍스트 프롬프트 필요 |
| 단일 참조 이미지 | 텍스트 프롬프트 필요 |

---

## 충돌 관계

| 매개변수 A | 매개변수 B | 설명 |
|--------|--------|------|
| `draft` 속도 | `hd` 품질 | 드래프트 모드는 HD와 함께 사용할 수 없음 |
| `draft` 속도 | 이미지 프롬프트 / `--sref` / `--oref` | 이미지 참조는 `fast` 필요 |
| 순수 이미지 프롬프트 | `--stylize` / `--exp` | 텍스트 없이는 호환되지 않음 |
| `--exp` > 25 | `--stylize` | 높은 `--exp`는 `--stylize`를 억제할 수 있음. 함께 사용할 때는 `--exp`를 낮추세요 |

## V8.1에서 지원하지 않는 매개변수

| 매개변수 | 설명 |
|------|------|
| `--q` / `--quality` | 본 인터페이스에서는 정밀도 플래그를 제공하지 않음 (해상도는 최상위 `quality` 매개변수 사용) |
| `--no` | 본 인터페이스에서는 부정 프롬프트를 제공하지 않음 |
| `--weird` / `--w` | 미지원 |
| `--tile` | 미지원 |
| `--sv` | 스타일 버전이 고정되어 제공되지 않음 |
| `--stop` | 미지원 |
| `--cref` / `--cw` | 대신 `--oref` 오브젝트 참조 사용 |
| `--relax` | 미지원 |
| `--repeat` / `--r` | API에서 미지원 |
| `--p` (personalize) | API에서 미지원 |
| `{}` 열거 | API에서 순열 프롬프트 미지원 |
| `--stealth` / `--public` | API에서 지원하지 않음 (이미지는 공개되지 않음) |
| `--niji` | 이는 다른 모델이며 V8.1이 아님 |
| 다중 프롬프트 `::` | 미지원 |

## 프롬프트 작성 규칙

### 기본 구조

```
[이미지URL] 텍스트 설명 --매개변수1 값1 --매개변수2 값2
```

### 작성 규칙

- 매개변수는 텍스트 프롬프트의 **끝**에 배치
- `--` 앞에는 반드시 **공백**이 필요
- 매개변수 안에 구두점을 사용하지 않음
- 매개변수 뒤에 텍스트를 작성하지 않음

### 예시

<CodeGroup>
```text 올바름
a yellow Persian cat --ar 1:2 --s 500
```

```text 잘못됨
a yellow Persian cat--ar 2:3          ← -- 앞에 공백 없음
a yellow Persian cat - - ar 2:3       ← -- 사이에 공백 있음
a yellow Persian cat --ar 2:3, --s 50 ← 매개변수에 구두점 있음
a yellow Persian cat --ar 2:3 more description ← 매개변수 뒤에 텍스트 있음
```
</CodeGroup>

## 콘텐츠 검토 안내

<Note>
Midjourney에는 콘텐츠 검토 시스템이 내장되어 있습니다. 생성된 이미지 중 일부가 검토에 의해 필터링된 경우, 해당 요청에서 소비된 크레딧은 환불되지 않습니다. 프롬프트가 콘텐츠 가이드라인을 준수하는지 확인해 주세요.
</Note>
