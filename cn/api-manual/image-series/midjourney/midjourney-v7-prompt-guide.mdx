## 参数总览

| 参数 | 写法 | 类型 | 取值范围 | 默认值 | 说明 |
|------|------|------|---------|--------|------|
| 宽高比 | `--ar W:H` | 整数比 | 任意正整数比 | 1:1 | 图像宽高比 |
| 混沌 | `--c N` | int | 0 - 100 | 0 | 生成结果的多样性 |
| 否定提示 | `--no 词1,词2` | 关键词 | — | — | 排除不想出现的元素 |
| 质量 | `--q N` | 枚举 | 1, 2, 4 | 1 | 生成精细度，**影响费用** |
| 种子 | `--seed N` | int | 0 - 4294967295 | 随机 | 固定种子可复现结果 |
| 风格化 | `--s N` | int | 0 - 1000 | 100 | 艺术风格强度 |
| 怪异 | `--w N` | int | 0 - 3000 | 0 | 非常规/超现实程度 |
| 实验参数 | `--exp N` | int | 0 - 100 | 0 | 美学效果（V7 专属） |
| 原始模式 | `--raw` | 标志 | — | 关闭 | 禁用默认美化 |
| 平铺 | `--tile` | 标志 | — | 关闭 | 无缝重复图案 |
| 图像权重 | `--iw N` | float | 0 - 3 | 1 | 垫图影响力 |
| 风格参考 | `--sref [URL]` | URL | — | — | 匹配视觉风格 |
| 风格权重 | `--sw N` | int | 0 - 1000 | 100 | 风格参考强度 |
| 风格版本 | `--sv N` | 枚举 | 1, 2, 3, 4, 5, 6 | 4 | 风格算法版本 |
| 万物引用 | `--oref [URL]` | URL | — | — | 放置角色/物体（V7 专属），**费用翻倍** |
| 万物引用权重 | `--ow N` | int | 1 - 1000 | 100 | 万物引用强度 |
| 草图模式 | `--draft` | 标志 | — | 关闭 | 快 10 倍、费用减半（V7 专属），通过 API `model_params.speed` 控制 |

<Note>
速度模式（draft / fast / turbo）通过 API 的 `model_params.speed` 字段控制，无需在 prompt 中手写 `--draft` / `--fast` / `--turbo`。
</Note>

## 基础参数

### 宽高比 `--ar`

设置图像的宽高比，仅支持整数比。

```
一只猫 --ar 16:9
```

常见值：`1:1`、`4:3`、`3:2`、`16:9`、`9:16`、`2:3`

### 混沌 `--chaos` / `--c`

控制生成结果的多样性。值越高，4 张图之间差异越大。

```
一只猫 --c 50
```

| 范围 | 效果 |
|------|------|
| 0 | 4 张图非常相似（默认） |
| 1-50 | 适度多样性 |
| 50-100 | 高度随机，适合创意探索 |

### 否定提示 `--no`

排除不想出现的元素，等价于 -0.5 权重。支持多个关键词。

```
生机勃勃的郁金香田 --no 红色,阴影
```

<Warning>
V7 不支持 `::` 多提示词语法，只能使用 `--no` 排除不需要的元素。
</Warning>

### 质量 `--quality` / `--q`

控制生成的精细度。

```
一只猫 --q 2
```

| 值 | 说明 |
|----|------|
| 1 | 默认，优化了手部连贯性 |
| 2 | 使用优化前的 V7 模型 |
| 4 | 实验模式（与 `--oref` 不兼容） |

<Info>
质量值越高，GPU 消耗越大，费用相应增加。
</Info>

### 种子 `--seed`

固定初始随机状态，用于对比测试。

```
一只猫 --seed 23453422
```

- 范围：0 - 4294967295
- 相同种子 + 相同提示词 → 相似（非完全相同）的结果
- 不适用于 turbo 速度模式

### 风格化 `--stylize` / `--s`

调节写实与艺术之间的平衡。

```
一只猫 --s 500
```

| 范围 | 效果 |
|------|------|
| 0-250 | 写实风格，忠实于提示词 |
| 250-750 | 平衡 |
| 750-1000 | 强烈艺术性，大胆色彩构图 |

### 怪异 `--weird` / `--w`

引入非常规和超现实元素。

```
太空风景 --w 800
```

| 范围 | 效果 |
|------|------|
| 10-300 | 轻微怪异 |
| 300-1000 | 明显非传统元素 |
| 1000-3000 | 极端超现实和抽象 |

### 实验参数 `--exp`（V7 专属）

与 `--stylize` 类似但可叠加，生成更详细、动态、创意的图像。

```
一只猫 --exp 25
```

- 推荐值：5、10、25、50、100
- 5-50 效果变化明显，50-100 变化较小
- 超过 25-50 时可能覆盖 `--stylize` 的效果

### 原始模式 `--raw`

禁用默认美化，更严格地遵循提示词细节。适用于追求真实感或精确控制的场景。

```
产品照片 --raw
```

### 平铺 `--tile`

生成可无缝重复的图案（上下左右四向无缝连接），适用于纹理、壁纸、织物设计。

```
砖墙纹理 --tile
```

## 图像引用参数

### 图像提示（垫图）

在提示词**开头**放置图片 URL，用图片影响生成内容。

```
https://example.com/ref.jpg 文本描述 --iw 1.5
```

**有效组合规则：**

| 组合方式 | 是否有效 |
|---------|---------|
| 1 张图 + 无文字 | ❌ 无效（会报错） |
| 1 张图 + 文字描述 | ✅ 有效 |
| 2+ 张图 + 无文字 | ✅ 有效 |
| 2+ 张图 + 文字描述 | ✅ 有效 |

- 最多 20 张垫图
- 支持格式：`.png`, `.gif`, `.webp`, `.jpg`, `.jpeg`

### 图像权重 `--iw`

控制垫图对结果的影响力。范围 0 - 3，默认 1，值越高越接近参考图。

```
https://example.com/ref.jpg 水彩风景 --iw 2.0
```

### 风格参考 `--sref`

匹配参考图的视觉风格（颜色、纹理、光照），不复制内容。**必须配合文本提示词使用**。

```
一只猫 --sref https://example.com/style.jpg
```

- 支持多张：`--sref URL1 URL2`
- 支持分配权重：`--sref URL1::2 URL2::1`（注意：此处 `::` 是 sref 参数内部的权重语法，与 V7 不支持的多提示词 `::` 分隔符是不同的概念）
- 支持随机风格：`--sref random`（生成后返回数字代码，可重用）
- 最多 20 个 sref

### 风格权重 `--sw`

控制风格参考的影响程度。范围 0 - 1000，默认 100。

```
一只猫 --sref https://example.com/style.jpg --sw 500
```

### 风格版本 `--sv`

切换风格参考的算法版本。

| 版本 | 特点 |
|------|------|
| 1 | 原始算法，更"有氛围" |
| 2 | 第二次迭代 |
| 3 | v1 更新版 |
| 4 | v2 更新版（**默认**） |
| 5 | V7 新增 |
| 6 | V7 新增 |

### 万物引用 `--oref`（V7 专属）

将参考图中的角色、物体、车辆或非人类生物放入新图中。**必须配合文本提示词使用**。

```
一只手拿魔法棒的小鸭子 --oref https://example.com/duck.jpg
```

<Warning>
使用万物引用将消耗 2 倍 GPU 时间，费用翻倍。且仅能使用一张参考图片。
</Warning>

### 万物引用权重 `--ow`（V7 专属）

范围 1 - 1000，默认 100。建议保持在 400 以下，否则结果可能不可预测。高 `--stylize` 值时可能需要更高 `--ow` 以补偿。

```
小鸭子 --oref https://example.com/duck.jpg --ow 300
```

## 速度模式

速度模式通过 API 的 `model_params.speed` 字段控制，系统自动注入到提示词。

| 模式 | 说明 | 费用 |
|------|------|------|
| `draft` | 快 10 倍，质量较低，可后续用 [mj-v7-enhance](/cn/api-manual/image-series/midjourney/mj-v7-enhance) 增强。**仅限生图接口** | 最低 |
| `fast` | 标准质量（默认） | 标准 |
| `turbo` | 最高速度 | 约为 fast 的 2 倍 |

## 依赖关系

以下参数必须搭配其他参数才能生效：

| 参数 | 前置依赖 |
|------|---------|
| `--sv`、`--sw` | 需要 `--sref` |
| `--iw` | 需要图像提示（垫图） |
| `--ow` | 需要 `--oref` |
| `--sref`、`--oref` | 需要文本提示词 |
| 单张垫图 | 需要文本提示词 |

---

## 冲突关系

| 参数 A | 参数 B | 说明 |
|--------|--------|------|
| `--oref` | draft / turbo 速度 | 万物引用仅兼容 fast 模式 |
| `--oref` | `--q 4` | 质量 4 与万物引用不兼容 |
| `--oref` | inpaint/pan/outpaint | 万物引用的图片不能用于这些操作 |
| `--draft` | `--tile` | 草图模式不支持平铺 |
| 纯图像提示 | `--stylize` / `--weird` | 无文字时不兼容 |

<Tip>
`--exp` 值超过 25-50 时会压制 `--stylize` 的效果。建议组合使用时 exp 取较低值（5-25）。
</Tip>

## V7 不支持的参数

| 参数 | 说明 |
|------|------|
| `--stop` | V7 不支持 |
| `--cref` / `--cw` | 改用 `--oref` 万物引用 |
| `--relax` | V7 不支持慢速模式 |
| `--repeat` / `--r` | API 暂不支持 |
| `--p` (personalize) | API 暂不支持 |
| `{}` 枚举 | API 暂不支持排列提示词 |
| `--stealth` / `--public` | API 暂不支持（图片均不对外开放） |
| 多提示词 `::` | V7 不支持 |

## 提示词格式规范

### 基本结构

```
[图片URL] 文本描述 --参数1 值1 --参数2 值2
```

### 书写规则

- 参数放在文本提示**末尾**
- `--` 前必须有**空格**
- 参数中不使用标点符号
- 参数后面不能再写文本

### 示例

<CodeGroup>
```text 正确
一只黄色波斯猫 --ar 1:2 --s 500
```

```text 错误
一只黄色波斯猫--ar 2:3          ← --前缺空格
一只黄色波斯猫 - - ar 2:3       ← --间有空格
一只黄色波斯猫 --ar 2:3, --s 50 ← 参数中有标点
一只黄色波斯猫 --ar 2:3 更多描述 ← 参数后有文本
```
</CodeGroup>

## 内容审核说明

<Note>
Midjourney 内置内容审核机制。如果生成的部分图像触发了审核过滤，该次请求已消耗的积分将无法退还，请留意提示词的内容合规性。
</Note>
