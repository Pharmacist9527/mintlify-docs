## 参数总览

| 参数 | 写法 | 类型 | 取值范围 | 默认值 | 说明 |
|------|------|------|---------|--------|------|
| 宽高比 | `--ar W:H` | 整数比 | 任意正整数比 | 1:1 | 图像宽高比 |
| 混沌 | `--c N` | int | 0 - 100 | 0 | 生成结果的多样性 |
| 种子 | `--seed N` | int | 0 - 4294967295 | 随机 | 固定种子可复现结果 |
| 风格化 | `--s N` | int | 0 - 1000 | 100 | 艺术风格强度 |
| 实验参数 | `--exp N` | int | 0 - 100 | 0 | 美学效果，可与风格化叠加 |
| 原始模式 | `--raw` | 标志 | — | 关闭 | 禁用默认美化 |
| 图像权重 | `--iw N` | float | 0 - 3 | 1 | 垫图影响力 |
| 风格参考 | `--sref [URL]` | URL | — | — | 匹配视觉风格 |
| 风格权重 | `--sw N` | int | 0 - 1000 | 100 | 风格参考强度 |
| 万物引用 | `--oref [URL]` | URL | — | — | 将角色/物体放入画面 |
| 万物引用权重 | `--ow N` | int | 1 - 1000 | 100 | 万物引用强度 |

<Note>
有两项设置**不**写入提示词，而是通过 API 控制（写在提示词中不会生效）：

- **速度**（`draft` / `fast` / `turbo`）→ `model_params.speed`
- **输出质量**（`standard` / `hd`）→ 顶层 `quality` 参数

`--v` / `--version` 锁定为 V8.1，`--niji` 不支持。详见下文 [速度模式](#speed-modes) 和 [输出质量](#output-quality)。
</Note>

## 基础参数

### 宽高比 `--ar`

设置图像的宽高比，仅支持整数比。

```
一只猫 --ar 16:9
```

常见值：`1:1`、`4:3`、`3:2`、`16:9`、`9:16`、`2:3`。极端比例可能产生不可预测的结果。

### 混沌 `--chaos` / `--c`

控制生成结果的多样性。值越高，4 张图之间差异越大。

```
一只猫 --c 50
```

| 范围 | 效果 |
|------|------|
| 0 | 4 张图非常相似（默认） |
| 1-50 | 适度多样性 |
| 50-100 | 高度随机，适合创意探索 |

### 风格化 `--stylize` / `--s`

调节写实与艺术之间的平衡。

```
一只猫 --s 500
```

| 范围 | 效果 |
|------|------|
| 0-250 | 写实风格，忠实于提示词 |
| 250-750 | 平衡 |
| 750-1000 | 强烈艺术性，大胆色彩构图 |

### 实验参数 `--exp`

与 `--stylize` 类似但可叠加，生成更详细、动态、创意的图像。

```
一只猫 --exp 25
```

- 推荐值：5、10、25、50、100
- 5-50 效果变化明显，50-100 变化较小
- 超过 25-50 时可能覆盖 `--stylize` 的效果

### 原始模式 `--raw`

禁用默认美化，更严格地遵循提示词细节。适用于追求真实感或精确控制的场景。

```
产品照片 --raw
```

### 种子 `--seed`

固定初始随机状态，用于对比测试。

```
一只猫 --seed 23453422
```

- 范围：0 - 4294967295
- 相同种子 + 相同提示词 → 相似（非完全相同）的结果
- 不适用于 turbo 速度模式

## 图像引用参数

### 图像提示（垫图）

在提示词**开头**放置图片 URL，用图片影响生成内容。

```
https://example.com/ref.jpg 文本描述 --iw 1.5
```

**有效组合规则：**

| 组合方式 | 是否有效 |
|---------|---------|
| 1 张图 + 无文字 | ❌ 无效（会报错） |
| 1 张图 + 文字描述 | ✅ 有效 |
| 2+ 张图 + 无文字 | ✅ 有效 |
| 2+ 张图 + 文字描述 | ✅ 有效 |

- 最多 20 张垫图
- 支持格式：`.png`、`.gif`、`.webp`、`.jpg`、`.jpeg`
- 不支持 `draft` 速度（请改用 `fast`）

### 图像权重 `--iw`

控制垫图对结果的影响力。范围 0 - 3，默认 1，值越高越接近参考图。

```
https://example.com/ref.jpg 水彩风景 --iw 2.0
```

### 风格参考 `--sref`

匹配参考图的视觉风格（颜色、纹理、光照），不复制内容。**必须配合文本提示词使用**。

```
一只猫 --sref https://example.com/style.jpg
```

- 支持多张：`--sref URL1 URL2`
- 支持随机风格：`--sref random`（生成后返回数字代码，可重用）
- 最多 20 个 sref

### 风格权重 `--sw`

控制风格参考的影响程度。范围 0 - 1000，默认 100。

```
一只猫 --sref https://example.com/style.jpg --sw 500
```

### 万物引用 `--oref`

将参考图中的角色、物体、车辆或非人类生物放入新图中。**必须配合文本提示词使用**。

```
一只手拿魔法棒的小鸭子 --oref https://example.com/duck.jpg
```

<Warning>
仅能使用一张万物引用图片，且启用后生成耗时会更长。
</Warning>

### 万物引用权重 `--ow`

范围 1 - 1000，默认 100。建议保持在 400 以下，否则结果可能不可预测。高 `--stylize` 值时可能需要更高 `--ow` 以补偿。

```
小鸭子 --oref https://example.com/duck.jpg --ow 300
```

## 速度模式

速度模式通过 API 的 `model_params.speed` 字段控制（请勿在提示词中写 `--draft` / `--fast` / `--turbo`）。

| 模式 | 说明 | 费用 |
|------|------|------|
| `draft` | 单次运行返回 24 张轻量级 0.5K 草图。**仅限生图接口** | 与 fast 同倍率 |
| `fast` | 标准模式（默认），每次生成 4 张图 | 标准 |
| `turbo` | 优先 / 最高速度，每次生成 4 张图 | 约为 fast 的 2 倍 |

<Note>
与 V7 不同，V8.1 的 `draft` 模式**不**减半收费——它与 `fast` 使用相同倍率。草图模式单次运行返回 24 张小图，挑出喜欢的再以完整质量重跑。草图模式不能与 `hd` 质量组合使用。
</Note>

## 输出质量

输出分辨率通过顶层 `quality` 参数控制，**而非**提示词标志。请勿在提示词中写 `--hd`。

| 值 | 分辨率 | 费用 |
|----|--------|------|
| `standard` | 标准分辨率（默认） | 1 倍 |
| `hd` | 原生 2K 高清输出 | 1.5 倍 |

- 质量倍率与速度倍率是**相乘**关系——例如 `turbo` + `hd` = 2 × 1.5 = 基准的 3 倍。
- `hd` 与 `draft` 速度**不兼容**。

## 依赖关系

以下参数必须搭配其他参数才能生效：

| 参数 | 前置依赖 |
|------|---------|
| `--sw` | 需要 `--sref` |
| `--ow` | 需要 `--oref` |
| `--iw` | 需要图像提示（垫图） |
| `--sref`、`--oref` | 需要文本提示词 |
| 单张垫图 | 需要文本提示词 |

---

## 冲突关系

| 参数 A | 参数 B | 说明 |
|--------|--------|------|
| `draft` 速度 | `hd` 质量 | 草图模式不能与 HD 组合 |
| `draft` 速度 | 图像提示 / `--sref` / `--oref` | 图像引用需要 `fast`（或 `turbo`） |
| 纯图像提示 | `--stylize` / `--exp` | 无文字时不兼容 |
| `--exp` > 25 | `--stylize` | 高 `--exp` 可能压制 `--stylize`；组合使用时请调低 `--exp` |

## V8.1 不支持的参数

| 参数 | 说明 |
|------|------|
| `--q` / `--quality` | 本接口不开放精细度标志（分辨率请改用顶层 `quality` 参数） |
| `--no` | 本接口不开放否定提示 |
| `--weird` / `--w` | 不支持 |
| `--tile` | 不支持 |
| `--sv` | 风格版本已锁定，不开放 |
| `--stop` | 不支持 |
| `--cref` / `--cw` | 改用 `--oref` 万物引用 |
| `--relax` | 不支持 |
| `--repeat` / `--r` | API 暂不支持 |
| `--p` (personalize) | API 暂不支持 |
| `{}` 枚举 | API 暂不支持排列提示词 |
| `--stealth` / `--public` | API 暂不支持（图片均不对外开放） |
| `--niji` | 这是另一个模型，并非 V8.1 |
| 多提示词 `::` | 不支持 |

## 提示词格式规范

### 基本结构

```
[图片URL] 文本描述 --参数1 值1 --参数2 值2
```

### 书写规则

- 参数放在文本提示**末尾**
- `--` 前必须有**空格**
- 参数中不使用标点符号
- 参数后面不能再写文本

### 示例

<CodeGroup>
```text 正确
a yellow Persian cat --ar 1:2 --s 500
```

```text 错误
a yellow Persian cat--ar 2:3          ← --前缺空格
a yellow Persian cat - - ar 2:3       ← --间有空格
a yellow Persian cat --ar 2:3, --s 50 ← 参数中有标点
a yellow Persian cat --ar 2:3 more description ← 参数后有文本
```
</CodeGroup>

## 内容审核说明

<Note>
Midjourney 内置内容审核机制。如果生成的部分图像触发了审核过滤，该次请求已消耗的积分将无法退还，请留意提示词的内容合规性。
</Note>
