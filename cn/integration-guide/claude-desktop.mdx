## 概述

![Claude Desktop](/images/integration-guide/claude-desktop-overview.png)

Claude Desktop 是 Anthropic 官方桌面客户端，适合在本地电脑上直接使用 Claude。

本文介绍如何将 Claude Desktop 连接到 **EvoLink API**，并通过 EvoLink 访问兼容 Anthropic Messages API 的模型能力。

**本文包含：**
- 安装 Claude Desktop
- 开启开发者模式
- 配置 EvoLink 连接参数
- 重启并选择模型

## 使用前准备

在开始配置之前，请先准备好 EvoLink API Key。

### 获取 EvoLink API Key

- 登录 [EvoLink 控制台](https://evolink.ai/dashboard)
- 在控制台中找到 API Keys，点击"创建新Key"按钮，然后复制生成的 Key
- API Key 通常以 `sk-` 开头，请妥善保存

## 第一步：下载 Claude Desktop

访问 [Claude Desktop 下载页](https://claude.com/download)，根据您的操作系统下载并安装对应版本。

<Note>
**提示：** 如果您还没有安装 Claude Desktop，请先完成安装再继续后续配置。
</Note>

<Frame>
![Claude Desktop 下载页](/images/integration-guide/claude-desktop-download-page.png)
</Frame>

### 安装后检查

安装完成后，建议先确认 Claude Desktop 可以正常启动。

## 第二步：开启开发者模式

启动 Claude Desktop 后，点击菜单栏 **Help → Troubleshooting**，在弹出的面板中启用开发者模式（Enable Developer Mode）。

启用后，菜单栏会出现 **Developer** 入口，用于后续配置第三方推理。

<Note>
**提示：** 如果菜单中没有出现 Developer 入口，请确认您的 Claude Desktop 版本支持开发者模式和第三方推理配置。
</Note>

<Frame>
![Claude Desktop 帮助菜单中的 Troubleshooting](/images/integration-guide/claude-desktop-help-menu.png)
</Frame>

<Frame>
![Claude Desktop 开发者菜单](/images/integration-guide/claude-desktop-developer-menu.png)
</Frame>

## 第三步：打开第三方推理配置

在开发者菜单中，进入第三方推理配置面板，按下表填写：

| 字段 | 说明 |
|---|---|
| Gateway base URL | EvoLink 的网关地址，建议填写 `https://direct.evolink.ai` |
| Gateway API key | 直接填写您的 EvoLink API Key 本身，不需要加 `Bearer` 前缀 |
| Gateway auth scheme | `bearer` |
| Model list | 可以不填。不填写时，Claude Desktop 会展示该 API Key 有权限的所有模型。您也可以直接跳到文末的 [支持的模型](#supported-claude-models) 表格 |

<Note>
**说明：** 如果您的 Claude Desktop 版本中字段名称略有不同，请以实际界面为准，但含义应保持一致。
</Note>

<Frame>
![Claude Desktop 第三方推理配置面板](/images/integration-guide/claude-desktop-third-party-inference.png)
</Frame>

## 第四步：保存并重启

保存配置后，完全退出 Claude Desktop，再重新启动。

<Frame>
![Claude Desktop 重启确认弹窗](/images/integration-guide/claude-desktop-relaunch-dialog.png)
</Frame>

重启后，Claude Desktop 应该会使用新的第三方推理配置。

**预期结果：** 启动后可以看到可用模型列表，且请求会通过 EvoLink 转发。

## 第五步：选择模型

重启后，在模型选择区域确认可用模型是否正常出现。

## 工作方式

Claude Desktop 通过 Anthropic 兼容的 Gateway 与 EvoLink 交互：

1. Claude Desktop 将请求发送到您配置的 Gateway 地址。
2. EvoLink 接收请求并转发到对应的模型服务。
3. 返回结果后，Claude Desktop 继续在本地展示对话结果。

这个流程不需要额外安装本地代理程序。

## 常见问题

### 开发者菜单没有出现

请确认您已开启开发者模式，并且已完整重启过 Claude Desktop。

### 配置后无法连接

请检查以下内容：

- Base URL 是否填写正确
- API Key 是否有效
- 认证方式是否为 `bearer`
- 是否已经完全退出并重新打开 Claude Desktop

### 没有看到模型

请确认账号和网络状态正常，再检查第三方推理配置是否已成功保存。

### `Model list` 要不要填？

可以不填。不填写时，Claude Desktop 会展示该 API Key 有权限的所有模型。

### 如果要手动指定模型，应该填什么？

填写上方"支持的模型"表格中的模型 ID，多个模型按 Claude Desktop 支持的格式填写即可。
