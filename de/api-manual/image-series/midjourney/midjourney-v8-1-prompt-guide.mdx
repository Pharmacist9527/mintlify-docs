## Parameterübersicht

| Parameter | Schreibweise | Typ | Wertebereich | Standard | Beschreibung |
|-----------|-------------|-----|-------------|----------|-------------|
| Seitenverhältnis | `--ar W:H` | Ganzzahlverhältnis | Beliebiges positives Ganzzahlverhältnis | 1:1 | Bildseitenverhältnis |
| Chaos | `--c N` | int | 0 - 100 | 0 | Vielfalt der generierten Ergebnisse |
| Seed | `--seed N` | int | 0 - 4294967295 | Zufällig | Seed fixieren für reproduzierbare Ergebnisse |
| Stilisierung | `--s N` | int | 0 - 1000 | 100 | Künstlerische Stilintensität |
| Experimentell | `--exp N` | int | 0 - 100 | 0 | Ästhetischer Effekt, stapelbar mit Stilisierung |
| Raw-Modus | `--raw` | Flag | — | Aus | Standard-Verschönerung deaktivieren |
| Bildgewicht | `--iw N` | float | 0 - 3 | 1 | Einfluss des Bildprompts |
| Stilreferenz | `--sref [URL]` | URL | — | — | Visuellen Stil abgleichen |
| Stilgewicht | `--sw N` | int | 0 - 1000 | 100 | Stärke der Stilreferenz |
| Objektreferenz | `--oref [URL]` | URL | — | — | Charakter/Objekt in die Szene platzieren |
| Objektreferenz-Gewicht | `--ow N` | int | 1 - 1000 | 100 | Stärke der Objektreferenz |

<Note>
Zwei Einstellungen werden **nicht** in den Prompt geschrieben, sondern stattdessen über die API gesteuert (im Prompt geschrieben haben sie keine Wirkung):

- **Geschwindigkeit** (`draft` / `fast`) → `model_params.speed`
- **Ausgabequalität** (`standard` / `hd`) → Top-Level-Parameter `quality`

`--v` / `--version` ist auf V8.1 festgelegt und `--niji` wird nicht unterstützt. Siehe [Geschwindigkeitsmodi](#speed-modes) und [Ausgabequalität](#output-quality) weiter unten.
</Note>

## Grundlegende Parameter

### Seitenverhältnis `--ar`

Legt das Bildseitenverhältnis fest. Es werden nur Ganzzahlverhältnisse unterstützt.

```
eine Katze --ar 16:9
```

Gängige Werte: `1:1`, `4:3`, `3:2`, `16:9`, `9:16`, `2:3`. Extreme Verhältnisse können unvorhersehbare Ergebnisse erzeugen.

### Chaos `--chaos` / `--c`

Steuert die Vielfalt der generierten Ergebnisse. Höhere Werte erzeugen größere Unterschiede zwischen den 4 Bildern.

```
eine Katze --c 50
```

| Bereich | Effekt |
|---------|--------|
| 0 | 4 Bilder sind sehr ähnlich (Standard) |
| 1-50 | Mäßige Vielfalt |
| 50-100 | Hochgradig zufällig, geeignet für kreative Erkundung |

### Stilisierung `--stylize` / `--s`

Passt die Balance zwischen Realismus und künstlerischem Stil an.

```
eine Katze --s 500
```

| Bereich | Effekt |
|---------|--------|
| 0-250 | Realistischer Stil, dem Prompt treu |
| 250-750 | Ausgewogen |
| 750-1000 | Starker künstlerischer Ausdruck, kräftige Farben und Komposition |

### Experimentell `--exp`

Ähnlich wie `--stylize`, aber stapelbar, erzeugt detailliertere, dynamischere und kreativere Bilder.

```
eine Katze --exp 25
```

- Empfohlene Werte: 5, 10, 25, 50, 100
- Spürbare Effektänderungen von 5-50, abnehmende Wirkung von 50-100
- Kann die Wirkung von `--stylize` über 25-50 überschreiben

### Raw-Modus `--raw`

Deaktiviert die Standard-Verschönerung für eine strengere Einhaltung der Prompt-Details. Geeignet für Fotorealismus oder präzise Kontrollszenarien.

```
Produktfoto --raw
```

### Seed `--seed`

Fixiert den anfänglichen Zufallszustand für Vergleichstests.

```
eine Katze --seed 23453422
```

- Bereich: 0 - 4294967295
- Gleicher Seed + gleicher Prompt → ähnliche (nicht identische) Ergebnisse

## Bildreferenz-Parameter

### Bildprompt

Platzieren Sie Bild-URLs am **Anfang** des Prompts, um den generierten Inhalt zu beeinflussen.

```
https://example.com/ref.jpg Textbeschreibung --iw 1.5
```

**Gültige Kombinationsregeln:**

| Kombination | Gültig? |
|-------------|---------|
| 1 Bild + kein Text | ❌ Ungültig (erzeugt Fehler) |
| 1 Bild + Textbeschreibung | ✅ Gültig |
| 2+ Bilder + kein Text | ✅ Gültig |
| 2+ Bilder + Textbeschreibung | ✅ Gültig |

- Maximal 20 Bildprompts
- Unterstützte Formate: `.png`, `.gif`, `.webp`, `.jpg`, `.jpeg`
- Nicht verfügbar mit `draft`-Geschwindigkeit (verwenden Sie `fast`)

### Bildgewicht `--iw`

Steuert den Einfluss des Bildprompts auf das Ergebnis. Bereich 0 - 3, Standard 1. Höhere Werte erzeugen Ergebnisse näher am Referenzbild.

```
https://example.com/ref.jpg Aquarelllandschaft --iw 2.0
```

### Stilreferenz `--sref`

Gleicht den visuellen Stil (Farben, Texturen, Beleuchtung) des Referenzbilds ab, ohne den Inhalt zu kopieren. **Muss mit einem Textprompt verwendet werden**.

```
eine Katze --sref https://example.com/style.jpg
```

- Mehrere unterstützt: `--sref URL1 URL2`
- Zufälliger Stil unterstützt: `--sref random` (gibt nach der Generierung einen Zahlencode zurück, wiederverwendbar)
- Maximal 20 Srefs

### Stilgewicht `--sw`

Steuert den Einfluss der Stilreferenz. Bereich 0 - 1000, Standard 100.

```
eine Katze --sref https://example.com/style.jpg --sw 500
```

### Objektreferenz `--oref`

Platziert Charaktere, Objekte, Fahrzeuge oder nicht-menschliche Kreaturen aus dem Referenzbild in ein neues Bild. **Muss mit einem Textprompt verwendet werden**.

```
ein Entchen mit einem Zauberstab --oref https://example.com/duck.jpg
```

<Warning>
Es kann nur ein Objektreferenzbild verwendet werden, und die Generierung dauert länger, wenn es aktiv ist.
</Warning>

### Objektreferenz-Gewicht `--ow`

Bereich 1 - 1000, Standard 100. Empfohlen unter 400 zu bleiben, da die Ergebnisse sonst unvorhersehbar sein können. Hohe `--stylize`-Werte erfordern möglicherweise einen höheren `--ow`-Wert zum Ausgleich.

```
Entchen --oref https://example.com/duck.jpg --ow 300
```

## Geschwindigkeitsmodi

Der Geschwindigkeitsmodus wird über das API-Feld `model_params.speed` gesteuert (schreiben Sie nicht `--draft` / `--fast` in den Prompt).

| Modus | Beschreibung | Kosten |
|-------|-------------|--------|
| `draft` | Liefert 24 leichtgewichtige 0,5K-Skizzenbilder in einem einzigen Durchlauf. Nur für Bildgenerierungs-Endpunkt. | Gleicher Multiplikator wie fast |
| `fast` | Standardmodus (Standard), 4 Bilder pro Generierung | Standard |

<Note>
Anders als bei V7 ist der `draft`-Modus von V8.1 **nicht** zum halben Preis – er verwendet denselben Multiplikator wie `fast`. Draft liefert 24 kleine Skizzen in einem Durchlauf; wählen Sie die gewünschten aus und führen Sie sie in voller Qualität erneut aus. Draft kann nicht mit `hd`-Qualität kombiniert werden.
</Note>

## Ausgabequalität

Die Ausgabeauflösung wird über den Top-Level-Parameter `quality` gesteuert, **nicht** über ein Prompt-Flag. Schreiben Sie nicht `--hd` in den Prompt.

| Wert | Auflösung | Kosten |
|------|-----------|--------|
| `standard` | Standardauflösung (Standard) | 1x |
| `hd` | Natives 2K-HD-Ausgabe | 1,5x |

- Der Qualitätsmultiplikator wird mit dem Geschwindigkeitsmultiplikator **kombiniert (multipliziert)**.
- `hd` ist **inkompatibel** mit `draft`-Geschwindigkeit.

## Abhängigkeiten

Die folgenden Parameter erfordern andere Parameter, um wirksam zu werden:

| Parameter | Voraussetzung |
|-----------|--------------|
| `--sw` | Erfordert `--sref` |
| `--ow` | Erfordert `--oref` |
| `--iw` | Erfordert Bildprompt |
| `--sref`, `--oref` | Erfordert Textprompt |
| Einzelner Bildprompt | Erfordert Textprompt |

---

## Konflikte

| Parameter A | Parameter B | Beschreibung |
|-------------|-------------|-------------|
| `draft` Geschwindigkeit | `hd` Qualität | Draft-Modus kann nicht mit HD kombiniert werden |
| `draft` Geschwindigkeit | Bildprompt / `--sref` / `--oref` | Bildreferenzen erfordern `fast` |
| Nur-Bild-Prompt | `--stylize` / `--exp` | Ohne Text inkompatibel |
| `--exp` > 25 | `--stylize` | Hohes `--exp` kann `--stylize` unterdrücken; senken Sie `--exp` bei Kombination |

| Parameter | Beschreibung |

| `--weird` / `--w` | Nicht unterstützt |

| `--sv` | Stilversion ist festgelegt; nicht bereitgestellt |
| `--stop` | Nicht unterstützt |
| `--cref` / `--cw` | Verwenden Sie stattdessen `--oref` Objektreferenz |
| `--relax` | Nicht unterstützt |
| `--repeat` / `--r` | Über API nicht unterstützt |
| `--p` (personalize) | Über API nicht unterstützt |

## Prompt-Format-Richtlinien

### Grundstruktur

```
[Bild-URL] Textbeschreibung --param1 Wert1 --param2 Wert2
```

### Schreibregeln

- Parameter stehen am **Ende** des Textprompts
- Ein **Leerzeichen** ist vor `--` erforderlich
- Keine Satzzeichen in Parametern verwenden
- Nach Parametern darf kein Text folgen

### Beispiele

<CodeGroup>
```text Richtig
a yellow Persian cat --ar 1:2 --s 500
```

```text Falsch
a yellow Persian cat--ar 2:3          ← fehlendes Leerzeichen vor --
a yellow Persian cat - - ar 2:3       ← Leerzeichen zwischen --
a yellow Persian cat --ar 2:3, --s 50 ← Satzzeichen in Parametern
a yellow Persian cat --ar 2:3 more description ← Text nach Parametern
```
</CodeGroup>

## Hinweis zur Inhaltsmoderation

<Note>
Midjourney verfügt über ein integriertes Moderationssystem. Wenn einige generierte Bilder durch die Moderation gefiltert werden, können die für diese Anfrage verbrauchten Credits nicht erstattet werden. Bitte achten Sie darauf, dass Ihre Prompts den Inhaltsrichtlinien entsprechen.
</Note>
