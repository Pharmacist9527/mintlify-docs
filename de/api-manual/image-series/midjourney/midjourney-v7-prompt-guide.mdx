## Parameterübersicht

| Parameter | Schreibweise | Typ | Wertebereich | Standard | Beschreibung |
|-----------|-------------|-----|-------------|----------|-------------|
| Seitenverhältnis | `--ar W:H` | Ganzzahlverhältnis | Beliebiges positives Ganzzahlverhältnis | 1:1 | Bildseitenverhältnis |
| Chaos | `--c N` | int | 0 - 100 | 0 | Vielfalt der generierten Ergebnisse |
| Negativprompt | `--no Wort1,Wort2` | Schlüsselwörter | — | — | Unerwünschte Elemente ausschließen |
| Qualität | `--q N` | Enum | 1, 2, 4 | 1 | Detailgrad der Generierung, **beeinflusst Kosten** |
| Seed | `--seed N` | int | 0 - 4294967295 | Zufällig | Seed fixieren für reproduzierbare Ergebnisse |
| Stilisierung | `--s N` | int | 0 - 1000 | 100 | Künstlerische Stilintensität |
| Weird | `--w N` | int | 0 - 3000 | 0 | Unkonventioneller/surrealer Grad |
| Experimentell | `--exp N` | int | 0 - 100 | 0 | Ästhetischer Effekt (V7-exklusiv) |
| Raw-Modus | `--raw` | Flag | — | Aus | Standard-Verschönerung deaktivieren |
| Kachel | `--tile` | Flag | — | Aus | Nahtloses Wiederholungsmuster |
| Bildgewicht | `--iw N` | float | 0 - 3 | 1 | Einfluss des Bildprompts |
| Stilreferenz | `--sref [URL]` | URL | — | — | Visuellen Stil abgleichen |
| Stilgewicht | `--sw N` | int | 0 - 1000 | 100 | Stärke der Stilreferenz |
| Stilversion | `--sv N` | Aufzählung | 1, 2, 3, 4, 5, 6 | 4 | Stilalgorithmus-Version |
| Objektreferenz | `--oref [URL]` | URL | — | — | Charakter/Objekt platzieren (V7-exklusiv), **verdoppelt Kosten** |
| Objektreferenz-Gewicht | `--ow N` | int | 1 - 1000 | 100 | Stärke der Objektreferenz |
| Entwurfsmodus | `--draft` | Flag | — | aus | 10x schneller, halbe Kosten (V7 exklusiv), über API `model_params.speed` gesteuert |

<Note>
Der Geschwindigkeitsmodus (draft / fast / turbo) wird über das API-Feld `model_params.speed` gesteuert. Es ist nicht nötig, `--draft` / `--fast` / `--turbo` manuell in den Prompt zu schreiben.
</Note>

## Grundlegende Parameter

### Seitenverhältnis `--ar`

Legt das Bildseitenverhältnis fest. Es werden nur Ganzzahlverhältnisse unterstützt.

```
eine Katze --ar 16:9
```

Gängige Werte: `1:1`, `4:3`, `3:2`, `16:9`, `9:16`, `2:3`

### Chaos `--chaos` / `--c`

Steuert die Vielfalt der generierten Ergebnisse. Höhere Werte erzeugen größere Unterschiede zwischen den 4 Bildern.

```
eine Katze --c 50
```

| Bereich | Effekt |
|---------|--------|
| 0 | 4 Bilder sind sehr ähnlich (Standard) |
| 1-50 | Mäßige Vielfalt |
| 50-100 | Hochgradig zufällig, geeignet für kreative Erkundung |

### Negativprompt `--no`

Schließt unerwünschte Elemente aus, entspricht einem Gewicht von -0,5. Unterstützt mehrere Schlüsselwörter.

```
lebhaftes Tulpenfeld --no rot,Schatten
```

<Warning>
V7 unterstützt nicht die `::` Multi-Prompt-Syntax. Sie können nur `--no` verwenden, um unerwünschte Elemente auszuschließen.
</Warning>

### Qualität `--quality` / `--q`

Steuert den Detailgrad der Generierung.

```
eine Katze --q 2
```

| Wert | Beschreibung |
|------|-------------|
| 1 | Standard, mit optimierter Handkohärenz |
| 2 | Verwendet das V7-Modell vor der Optimierung |
| 4 | Experimenteller Modus (inkompatibel mit `--oref`) |

<Info>
Höhere Qualitätswerte verbrauchen mehr GPU-Ressourcen und erhöhen die Kosten entsprechend.
</Info>

### Seed `--seed`

Fixiert den anfänglichen Zufallszustand für Vergleichstests.

```
eine Katze --seed 23453422
```

- Bereich: 0 - 4294967295
- Gleicher Seed + gleicher Prompt → ähnliche (nicht identische) Ergebnisse
- Nicht anwendbar im Turbo-Geschwindigkeitsmodus

### Stilisierung `--stylize` / `--s`

Passt die Balance zwischen Realismus und künstlerischem Stil an.

```
eine Katze --s 500
```

| Bereich | Effekt |
|---------|--------|
| 0-250 | Realistischer Stil, dem Prompt treu |
| 250-750 | Ausgewogen |
| 750-1000 | Starker künstlerischer Ausdruck, kräftige Farben und Komposition |

### Weird `--weird` / `--w`

Fügt unkonventionelle und surreale Elemente hinzu.

```
Weltraumlandschaft --w 800
```

| Bereich | Effekt |
|---------|--------|
| 10-300 | Leicht seltsam |
| 300-1000 | Merklich unkonventionelle Elemente |
| 1000-3000 | Extrem surreal und abstrakt |

### Experimentell `--exp` (V7-exklusiv)

Ähnlich wie `--stylize`, aber stapelbar, erzeugt detailliertere, dynamischere und kreativere Bilder.

```
eine Katze --exp 25
```

- Empfohlene Werte: 5, 10, 25, 50, 100
- Spürbare Effektänderungen von 5-50, abnehmende Wirkung von 50-100
- Kann die Wirkung von `--stylize` über 25-50 überschreiben

### Raw-Modus `--raw`

Deaktiviert die Standard-Verschönerung für eine strengere Einhaltung der Prompt-Details. Geeignet für Fotorealismus oder präzise Kontrollszenarien.

```
Produktfoto --raw
```

### Kachel `--tile`

Erzeugt nahtlos wiederholbare Muster (in alle vier Richtungen nahtlos), geeignet für Texturen, Tapeten und Stoffdesigns.

```
Backsteinmauer-Textur --tile
```

## Bildreferenz-Parameter

### Bildprompt

Platzieren Sie Bild-URLs am **Anfang** des Prompts, um den generierten Inhalt zu beeinflussen.

```
https://example.com/ref.jpg Textbeschreibung --iw 1.5
```

**Gültige Kombinationsregeln:**

| Kombination | Gültig? |
|-------------|---------|
| 1 Bild + kein Text | ❌ Ungültig (erzeugt Fehler) |
| 1 Bild + Textbeschreibung | ✅ Gültig |
| 2+ Bilder + kein Text | ✅ Gültig |
| 2+ Bilder + Textbeschreibung | ✅ Gültig |

- Maximal 20 Bildprompts
- Unterstützte Formate: `.png`, `.gif`, `.webp`, `.jpg`, `.jpeg`

### Bildgewicht `--iw`

Steuert den Einfluss des Bildprompts auf das Ergebnis. Bereich 0 - 3, Standard 1. Höhere Werte erzeugen Ergebnisse näher am Referenzbild.

```
https://example.com/ref.jpg Aquarelllandschaft --iw 2.0
```

### Stilreferenz `--sref`

Gleicht den visuellen Stil (Farben, Texturen, Beleuchtung) des Referenzbilds ab, ohne den Inhalt zu kopieren. **Muss mit einem Textprompt verwendet werden**.

```
eine Katze --sref https://example.com/style.jpg
```

- Mehrere unterstützt: `--sref URL1 URL2`
- Gewichtszuweisung unterstützt: `--sref URL1::2 URL2::1` (Hinweis: Das `::` hier ist die Gewichtungssyntax innerhalb des sref-Parameters, ein anderes Konzept als der Multi-Prompt `::` Separator, den V7 nicht unterstützt)
- Zufälliger Stil unterstützt: `--sref random` (gibt nach der Generierung einen Zahlencode zurück, wiederverwendbar)
- Maximal 20 Srefs

### Stilgewicht `--sw`

Steuert den Einfluss der Stilreferenz. Bereich 0 - 1000, Standard 100.

```
eine Katze --sref https://example.com/style.jpg --sw 500
```

### Stilversion `--sv`

Wechselt die Version des Stilreferenz-Algorithmus.

| Version | Eigenschaften |
|---------|--------------|
| 1 | Originalalgorithmus, mehr "atmosphärisch" |
| 2 | Zweite Iteration |
| 3 | Aktualisierte Version von v1 |
| 4 | Aktualisierte Version von v2 (**Standard**) |
| 5 | Neu in V7 |
| 6 | Neu in V7 |

### Objektreferenz `--oref` (V7-exklusiv)

Platziert Charaktere, Objekte, Fahrzeuge oder nicht-menschliche Kreaturen aus dem Referenzbild in ein neues Bild. **Muss mit einem Textprompt verwendet werden**.

```
ein Entchen mit einem Zauberstab --oref https://example.com/duck.jpg
```

<Warning>
Die Objektreferenz verbraucht 2x GPU-Zeit und verdoppelt die Kosten. Es kann nur ein Referenzbild verwendet werden.
</Warning>

### Objektreferenz-Gewicht `--ow` (V7-exklusiv)

Bereich 1 - 1000, Standard 100. Empfohlen unter 400 zu bleiben, da die Ergebnisse sonst unvorhersehbar sein können. Hohe `--stylize`-Werte erfordern möglicherweise einen höheren `--ow`-Wert zum Ausgleich.

```
Entchen --oref https://example.com/duck.jpg --ow 300
```

## Geschwindigkeitsmodi

Der Geschwindigkeitsmodus wird über das API-Feld `model_params.speed` gesteuert und automatisch in den Prompt eingefügt.

| Modus | Beschreibung | Kosten |
|-------|-------------|--------|
| `draft` | 10x schneller, geringere Qualität, kann später mit [mj-v7-enhance](/de/api-manual/image-series/midjourney/mj-v7-enhance) verbessert werden. **Nur für Bildgenerierungs-Endpunkt** | Niedrigste |
| `fast` | Standardqualität (Standard) | Standard |
| `turbo` | Maximale Geschwindigkeit | ~2x fast |

## Abhängigkeiten

Die folgenden Parameter erfordern andere Parameter, um wirksam zu werden:

| Parameter | Voraussetzung |
|-----------|--------------|
| `--sv`, `--sw` | Erfordert `--sref` |
| `--iw` | Erfordert Bildprompt |
| `--ow` | Erfordert `--oref` |
| `--sref`, `--oref` | Erfordert Textprompt |
| Einzelner Bildprompt | Erfordert Textprompt |

---

## Konflikte

| Parameter A | Parameter B | Beschreibung |
|-------------|-------------|-------------|
| `--oref` | draft / turbo Geschwindigkeit | Objektreferenz nur mit fast-Modus kompatibel |
| `--oref` | `--q 4` | Qualität 4 ist inkompatibel mit Objektreferenz |
| `--oref` | inpaint/pan/outpaint | Objektreferenz-Bilder können nicht für diese Operationen verwendet werden |
| `--draft` | `--tile` | Draft-Modus unterstützt keine Kachelung |
| Nur-Bild-Prompt | `--stylize` / `--weird` | Ohne Text inkompatibel |

<Tip>
Wenn `--exp` 25-50 überschreitet, wird die Wirkung von `--stylize` unterdrückt. Es wird empfohlen, niedrigere exp-Werte (5-25) zu verwenden, wenn sie kombiniert werden.
</Tip>

## Nicht unterstützte Parameter in V7

| Parameter | Beschreibung |
|-----------|-------------|
| `--stop` | In V7 nicht unterstützt |
| `--cref` / `--cw` | Verwenden Sie stattdessen `--oref` Objektreferenz |
| `--relax` | V7 unterstützt keinen Relax-Modus |
| `--repeat` / `--r` | Über API nicht unterstützt |
| `--p` (personalize) | Über API nicht unterstützt |
| `{}` Permutationen | Über API nicht unterstützt |
| `--stealth` / `--public` | Von der API nicht unterstützt (Bilder sind nicht öffentlich zugänglich) |
| Multi-Prompt `::` | In V7 nicht unterstützt |

## Prompt-Format-Richtlinien

### Grundstruktur

```
[Bild-URL] Textbeschreibung --param1 Wert1 --param2 Wert2
```

### Schreibregeln

- Parameter stehen am **Ende** des Textprompts
- Ein **Leerzeichen** ist vor `--` erforderlich
- Keine Satzzeichen in Parametern verwenden
- Nach Parametern darf kein Text folgen

### Beispiele

<CodeGroup>
```text Richtig
eine gelbe Perserkatze --ar 1:2 --s 500
```

```text Falsch
eine gelbe Perserkatze--ar 2:3          ← fehlendes Leerzeichen vor --
eine gelbe Perserkatze - - ar 2:3       ← Leerzeichen zwischen --
eine gelbe Perserkatze --ar 2:3, --s 50 ← Satzzeichen in Parametern
eine gelbe Perserkatze --ar 2:3 mehr Beschreibung ← Text nach Parametern
```
</CodeGroup>

## Hinweis zur Inhaltsmoderation

<Note>
Midjourney verfügt über ein integriertes Moderationssystem. Wenn einige generierte Bilder durch die Moderation gefiltert werden, können die für diese Anfrage verbrauchten Credits nicht erstattet werden. Bitte achten Sie darauf, dass Ihre Prompts den Inhaltsrichtlinien entsprechen.
</Note>
