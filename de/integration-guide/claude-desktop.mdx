## Überblick

![Claude Desktop](/images/integration-guide/claude-desktop-overview.png)

Claude Desktop ist die offizielle Desktop-App von Anthropic für die lokale Nutzung von Claude.

In dieser Anleitung wird beschrieben, wie Claude Desktop mit der **EvoLink API** verbunden wird, um über EvoLink Modelle zu nutzen, die mit der Anthropic Messages API kompatibel sind.

**Diese Anleitung umfasst:**
- Claude Desktop installieren
- Developer Mode aktivieren
- Die EvoLink-Verbindung konfigurieren
- Neu starten und ein Modell auswählen

## Voraussetzungen

Bereiten Sie vor dem Start Ihren EvoLink API Key vor.

### EvoLink API Key abrufen

- Melden Sie sich in der [EvoLink-Konsole](https://evolink.ai/dashboard) an
- Suchen Sie in der Konsole nach API Keys, klicken Sie auf **Create New Key** und kopieren Sie den generierten Key
- API-Keys beginnen meist mit `sk-`; bewahren Sie ihn sicher auf

## Schritt 1: Claude Desktop herunterladen

Öffnen Sie die [Claude Desktop-Downloadseite](https://claude.com/download) und installieren Sie die passende Version für Ihr Betriebssystem.

<Note>
**Tipp:** Wenn Claude Desktop noch nicht installiert ist, schließen Sie die Installation zuerst ab und fahren Sie dann fort.
</Note>

<Frame>
![Claude Desktop Download-Seite](/images/integration-guide/claude-desktop-download-page.png)
</Frame>

### Nach der Installation

Prüfen Sie nach der Installation, ob Claude Desktop normal startet.

## Schritt 2: Developer Mode aktivieren

Nach dem Start von Claude Desktop klicken Sie auf **Help → Troubleshooting** und aktivieren im Dialog den Developer Mode.

Danach erscheint ein **Developer**-Menüeintrag für die Konfiguration der Drittanbieter-Inferenz.

<Note>
**Hinweis:** Falls der Developer-Eintrag nicht angezeigt wird, prüfen Sie, ob Ihre Claude-Desktop-Version Developer Mode und Drittanbieter-Inferenz unterstützt.
</Note>

## Schritt 3: Drittanbieter-Inferenz öffnen

Öffnen Sie im Developer-Menü das Drittanbieter-Inferenz-Panel und füllen Sie es wie folgt aus:

| Feld | Beschreibung |
|---|---|
| Gateway base URL | Adresse des EvoLink-Gateways, empfohlen: `https://direct.evolink.ai` |
| Gateway API key | Geben Sie Ihren EvoLink API Key direkt ein, ohne das Präfix `Bearer` |
| Gateway auth scheme | `bearer` |
| Model list | Optional. Wenn leer, zeigt Claude Desktop alle Modelle an, auf die dieser API Key Zugriff hat. Sie können auch direkt zur Tabelle der [unterstützten Modelle](#supported-claude-models) springen |

<Note>
**Hinweis:** Wenn die Feldnamen in Ihrer Claude-Desktop-Version leicht abweichen, verwenden Sie die tatsächlichen Bezeichnungen in der Oberfläche, die Bedeutung sollte jedoch gleich bleiben.
</Note>

<Frame>
![Claude Desktop Drittanbieter-Inferenz](/images/integration-guide/claude-desktop-third-party-inference.png)
</Frame>

## Schritt 4: Speichern und neu starten

Speichern Sie die Konfiguration, beenden Sie Claude Desktop vollständig und starten Sie es erneut.

<Frame>
![Claude Desktop Neustart-Dialog](/images/integration-guide/claude-desktop-relaunch-dialog.png)
</Frame>

Nach dem Neustart sollte Claude Desktop die neue Drittanbieter-Inferenz-Konfiguration verwenden.

**Erwartetes Ergebnis:** Nach dem Start sehen Sie die verfügbare Modellliste und die Anfragen werden über EvoLink weitergeleitet.

## Schritt 5: Modell auswählen

Prüfen Sie nach dem Neustart im Modell-Auswahlbereich, ob die verfügbaren Modelle korrekt angezeigt werden.

## Funktionsweise

Claude Desktop kommuniziert über ein Anthropic-kompatibles Gateway mit EvoLink:

1. Claude Desktop sendet Anfragen an die konfigurierte Gateway-Adresse.
2. EvoLink empfängt die Anfrage und leitet sie an den passenden Modelldienst weiter.
3. Nach der Antwort zeigt Claude Desktop den Dialog weiterhin lokal an.

Ein zusätzlicher lokaler Proxy ist nicht erforderlich.

## FAQ

### Das Developer-Menü erscheint nicht

Stellen Sie sicher, dass Developer Mode aktiviert ist und Claude Desktop vollständig neu gestartet wurde.

### Nach der Konfiguration kann keine Verbindung hergestellt werden

Prüfen Sie Folgendes:

- Ist die Base URL korrekt?
- Ist der API Key gültig?
- Ist das Authentifizierungsschema `bearer`?
- Wurde Claude Desktop vollständig geschlossen und erneut geöffnet?

### Ich sehe keine Modelle

Stellen Sie sicher, dass Konto und Netzwerk normal funktionieren, und prüfen Sie anschließend, ob die Drittanbieter-Inferenz erfolgreich gespeichert wurde.

### Muss ich `Model list` ausfüllen?

Nein. Wenn das Feld leer bleibt, zeigt Claude Desktop alle Modelle an, auf die dieser API Key Zugriff hat.

### Was soll ich eintragen, wenn ich Modelle manuell festlegen möchte?

Verwenden Sie die Modell-IDs aus der obigen Tabelle "Unterstützte Modelle" und tragen Sie sie so ein, wie es Claude Desktop erwartet.
