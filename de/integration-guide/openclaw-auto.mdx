## Überblick

OpenClaw Manager ist ein visuelles Verwaltungstool, mit dem Sie OpenClaw-Instanzen schnell installieren, konfigurieren und verwalten können. Keine manuelle Konfigurationsbearbeitung erforderlich — nur 3 Schritte bis zur Bereitstellung.

Zwei Messaging-Kanäle werden unterstützt:

| | Telegram | Feishu (Lark) |
|---|---|---|
| Verbindung | Bot API (Long Polling) | WebSocket-Dauerverbindung |
| Erforderliche Zugangsdaten | Bot Token + Ihre User ID | App ID + App Secret |
| Plattform-Einrichtung | Nur einen Bot erstellen | Eine App erstellen + WebSocket-Abonnement aktivieren |
| Benutzerauthentifizierung | Gebunden an User ID — nur autorisierte Benutzer können chatten | Jeder Benutzer, der dem Bot schreiben kann, kann chatten |
| Geeignet für | Persönliche Nutzung, internationale Benutzer | Teamzusammenarbeit, Benutzer in China |

## Voraussetzungen

### 1. Node.js installieren

OpenClaw Manager erfordert Node.js 22 oder höher.

<Tabs>
  <Tab title="Windows">
    Besuchen Sie die [offizielle Node.js-Website](https://nodejs.org/de/download), laden Sie das Windows-Installationsprogramm (.msi-Datei) herunter und führen Sie es durch Doppelklick aus.

    Öffnen Sie nach der Installation PowerShell zur Überprüfung:

    ```bash
    node --version
    npm --version
    ```

    <Note>
    Es wird empfohlen, PowerShell als Administrator auszuführen, um Berechtigungsprobleme bei der späteren Installation zu vermeiden.
    </Note>
  </Tab>
  <Tab title="macOS">
    **Methode 1: Installationsprogramm verwenden**

    Besuchen Sie die [offizielle Node.js-Website](https://nodejs.org/de/download), laden Sie das macOS-Installationsprogramm (.pkg-Datei) herunter und führen Sie es durch Doppelklick aus.

    **Methode 2: Homebrew verwenden**

    ```bash
    brew install node
    ```

    Öffnen Sie nach der Installation das Terminal zur Überprüfung:

    ```bash
    node --version
    npm --version
    ```

    <Note>
    Wenn Sie bei der späteren Installation auf Berechtigungsprobleme stoßen, müssen Sie möglicherweise `sudo` vor dem Befehl hinzufügen.
    </Note>
  </Tab>
</Tabs>

### 2. Git installieren

OpenClaw ist auf Git für Versionsverwaltung und Plugin-Installation angewiesen.

<Tabs>
  <Tab title="Windows">
    Besuchen Sie die [offizielle Git-Website](https://git-scm.com/downloads), laden Sie das Windows-Installationsprogramm herunter, führen Sie es durch Doppelklick aus und schließen Sie die Installation mit den Standardoptionen ab.

    Öffnen Sie nach der Installation PowerShell zur Überprüfung:

    ```bash
    git --version
    ```
  </Tab>
  <Tab title="macOS">
    **Methode 1: Homebrew verwenden (empfohlen)**

    ```bash
    brew install git
    ```

    **Methode 2: Xcode Command Line Tools verwenden**

    ```bash
    xcode-select --install
    ```

    Öffnen Sie nach der Installation das Terminal zur Überprüfung:

    ```bash
    git --version
    ```
  </Tab>
</Tabs>

### 3. EvoLink API-Schlüssel erhalten

- Melden Sie sich im [EvoLink Dashboard](https://evolink.ai/dashboard) an
- Suchen Sie im Dashboard nach API Keys, klicken Sie auf die Schaltfläche "Neuen Schlüssel erstellen" und kopieren Sie den generierten Schlüssel
- Der API-Schlüssel beginnt normalerweise mit `sk-`, bewahren Sie ihn sicher auf

## Schritt 1: Herunterladen & Ausführen

Laden Sie die ausführbare Datei für Ihre Plattform von [Releases](https://github.com/Pharmacist9527/openclaw-manager/releases) herunter:

![GitHub Releases-Seite](/images/OpenClaw-Auto/github-releases.png)

![Ausführbare Datei herunterladen](/images/OpenClaw-Auto/github-repo.png)

| Plattform | Datei |
|----------|------|
| Windows | `openclaw-manager-win-x64.exe` |
| macOS (Intel) | `openclaw-manager-macos-x64` |
| macOS (Apple Silicon) | `openclaw-manager-macos-arm64` |
| Linux | `openclaw-manager-linux-x64` |

<Warning>
Stellen Sie sicher, dass [Node.js 22+](https://nodejs.org/de/download) installiert ist.
</Warning>

Windows-Benutzer klicken mit der rechten Maustaste und führen als Administrator aus. macOS-/Linux-Benutzer führen im Terminal mit `sudo` aus:

```bash
chmod +x ./openclaw-manager && sudo ./openclaw-manager
```

Das Skript prüft und installiert OpenClaw bei Bedarf, startet dann die lokale Web-Verwaltungsoberfläche und öffnet Ihren Browser.

![Neue Instanz erstellen](/images/OpenClaw-Auto/home-new-instance.png)

## Schritt 2: Instanz erstellen

### 2.1 — Basisinformationen

| Feld | Beschreibung |
|-------|-------------|
| Instanzname | Name für Ihre Instanz (z. B. `mybot`), muss auf diesem Rechner eindeutig sein |
| Modell | Wählen Sie ein Modell: Haiku 4.5 / Sonnet 4.5 / Opus 4.5 / Opus 4.6 |
| Kanal | Wählen Sie einen Kanal: Telegram oder Feishu |

![Basisinformationen](/images/OpenClaw-Auto/wizard-step1-basic-info.png)

### 2.2 — Zugangsdaten & Bereitstellung

Alle Kanäle erfordern einen **EvoLink API Key**.

<Tabs>
  <Tab title="Telegram">
    Geben Sie den **Bot Token** von @BotFather ein und klicken Sie dann auf **Deploy**.

    Das Skript führt automatisch folgende Schritte aus:
    1. Erstellen der OpenClaw-Konfigurationsdatei
    2. Schreiben der Modell- und API-Key-Einstellungen
    3. Konfigurieren und Aktivieren des Telegram-Plugins
    4. Installieren und Starten des Gateway-Dienstes

    ![Telegram-Zugangsdaten](/images/OpenClaw-Auto/wizard-step2-telegram-credentials.png)
  </Tab>
  <Tab title="Feishu">
    Geben Sie die **App ID** und das **App Secret** von der Feishu Open Platform ein und klicken Sie dann auf **Deploy**.

    Das Skript führt automatisch folgende Schritte aus:
    1. Erstellen der OpenClaw-Konfigurationsdatei
    2. Schreiben der Modell- und API-Key-Einstellungen
    3. Konfigurieren des Feishu-Plugins (App ID / App Secret eintragen) und Aktivieren
    4. Installieren und Starten des Gateway-Dienstes

    ![Feishu-Zugangsdaten](/images/OpenClaw-Auto/wizard-step2-credentials.png)
  </Tab>
</Tabs>

Warten Sie, bis der Fortschrittsbalken abgeschlossen ist, dann gelangen Sie automatisch zum nächsten Schritt.

### 2.3 — Verbinden

<Tabs>
  <Tab title="Telegram">
    Geben Sie Ihre **Telegram User ID** (die numerische ID von @userinfobot) ein und klicken Sie dann auf **Connect**.

    Das Skript schreibt diese ID in die `allowedUsers`-Konfiguration — nur dieser Benutzer kann mit dem Bot chatten. Nach der Verbindung werden Sie zur Startseite weitergeleitet.

    ![Telegram verbinden](/images/OpenClaw-Auto/wizard-step3-telegram-connect.png)
  </Tab>
  <Tab title="Feishu">
    Die Bereitstellung ist abgeschlossen. Kehren Sie nun zur Feishu Open Platform zurück, um das Ereignisabonnement zu konfigurieren.

    <Warning>
    Stellen Sie vor der Konfiguration des Ereignisabonnements sicher, dass Folgendes erledigt ist:
    - Die Feishu-Kanalkonfiguration ist abgeschlossen (siehe Bereitstellung in Schritt 2)
    - Das Gateway läuft (prüfen Sie mit `openclaw gateway status`)
    </Warning>

    Auf der Seite **Ereignisabonnement**:

    1. Wählen Sie **Dauerverbindung zum Empfangen von Ereignissen verwenden** (WebSocket-Modus)
    2. Ereignis hinzufügen: `im.message.receive_v1` (Nachrichten empfangen)

    <Warning>
    Wenn das Gateway nicht gestartet ist oder der Kanal nicht hinzugefügt wurde, können die Einstellungen für die Dauerverbindung nicht gespeichert werden.
    </Warning>

    ![Ereignisabonnement](/images/OpenClaw-Feishu/event-subscription.avif)

    3. Erstellen Sie unter „Versionsverwaltung & Freigabe" eine Version und reichen Sie diese zur Überprüfung ein

    <Note>
    Feishu-Apps erfordern eine Admin-Genehmigung, bevor sie verwendet werden können. Benutzerdefinierte Unternehmens-Apps werden in der Regel automatisch genehmigt.
    </Note>

    Klicken Sie auf **Done** — das Skript startet das Gateway neu, um die Feishu-WebSocket-Verbindung herzustellen. Nach der Weiterleitung zur Startseite können Sie dem Bot auf Feishu Nachrichten senden.

    ![Feishu verbinden](/images/OpenClaw-Auto/wizard-step3-feishu-connect.png)
  </Tab>
</Tabs>

## Instanzen verwalten

Nach der Bereitstellung zeigt die Startseite alle Instanzen an:

![Instanzverwaltung](/images/OpenClaw-Auto/home-instance-running.png)

| Status | Beschreibung |
|--------|-------------|
| 🟢 Grün | Gateway läuft |
| 🔴 Rot | Gateway gestoppt |

Jede Instanzkarte zeigt das verwendete Modell und den Kanal an. Verfügbare Aktionen:

- **Start / Stop**: Gateway-Dienst starten oder stoppen
- **Delete**: Instanz entfernen (stoppt das Gateway und bereinigt alle Konfigurationsdateien)

Klicken Sie auf **+ New Instance**, um mehrere Instanzen zu erstellen, die jeweils unabhängig auf einem anderen Port laufen.
