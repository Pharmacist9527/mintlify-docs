## Descripción General

OpenClaw Manager es una herramienta de gestión visual que te ayuda a instalar, configurar y gestionar instancias de OpenClaw rápidamente. No es necesario editar configuraciones manualmente — solo 3 pasos para desplegar.

Se admiten dos canales de mensajería:

| | Telegram | Feishu (Lark) |
|---|---|---|
| Conexión | Bot API (long polling) | Conexión persistente WebSocket |
| Credenciales necesarias | Bot Token + tu User ID | App ID + App Secret |
| Configuración de plataforma | Solo crear un Bot | Crear una app + habilitar suscripción WebSocket |
| Autenticación de usuario | Vinculado por User ID — solo usuarios autorizados pueden chatear | Cualquier usuario que pueda enviar mensajes al Bot puede chatear |
| Ideal para | Uso personal, usuarios internacionales | Colaboración en equipo, usuarios en China |

## Verificación del Entorno del Sistema

Antes de la instalación, se recomienda ejecutar la herramienta de verificación del entorno para asegurarse de que su sistema cumple con los requisitos de OpenClaw.

### Descargar Herramienta de Verificación

Descargue la herramienta de verificación para su plataforma desde [GitHub Releases](https://github.com/suuuuuu-1/openclaw-env-checker/releases):

| Plataforma | Nombre del archivo |
|------|--------|
| Windows | `openclaw-checker-win-x64.exe` |
| macOS (Intel) | `openclaw-checker-macos-x64` |
| macOS (Apple Silicon) | `openclaw-checker-macos-arm64` |
| Linux | `openclaw-checker-linux-x64` |

### Elementos de Verificación

La herramienta verifica automáticamente lo siguiente:

- ✅ Versión de Node.js (requiere >= 22.12.0)
- ✅ npm disponible
- ✅ Git disponible
- ✅ Conectividad de red (github.com, npmjs.org, evolink.ai)

![Ejemplo de verificación exitosa](/images/OpenClaw-Checker/check-success.png)

Si la verificación falla, la herramienta proporcionará sugerencias de corrección específicas.

## Requisitos Previos

### Comunes

- **Node.js 22+**: El script instala OpenClaw mediante npm si no está presente, por lo que se requiere Node.js
- **Clave API de EvoLink**: Regístrate en [evolink.ai](https://evolink.ai/dashboard) y obtén tu clave API

## Paso 1: Descargar y Ejecutar

Descarga el ejecutable para tu plataforma desde [Releases](https://github.com/Pharmacist9527/openclaw-manager/releases):

![Página de Releases en GitHub](/images/OpenClaw-Auto/github-releases.png)

![Descargar ejecutable](/images/OpenClaw-Auto/github-repo.png)

| Plataforma | Archivo |
|----------|------|
| Windows | `openclaw-manager-win-x64.exe` |
| macOS (Intel) | `openclaw-manager-macos-x64` |
| macOS (Apple Silicon) | `openclaw-manager-macos-arm64` |
| Linux | `openclaw-manager-linux-x64` |

<Warning>
Asegúrate de tener instalado [Node.js 22+](https://nodejs.org/es/download).
</Warning>

Los usuarios de Windows deben hacer clic derecho y ejecutar como administrador. Los usuarios de macOS / Linux deben ejecutar con `sudo` en la terminal:

```bash
chmod +x ./openclaw-manager && sudo ./openclaw-manager
```

El script verificará e instalará OpenClaw si es necesario, luego iniciará la interfaz web de gestión local y abrirá tu navegador.

![Crear nueva instancia](/images/OpenClaw-Auto/home-new-instance.png)

## Paso 2: Crear una Instancia

### 2.1 — Información Básica

| Campo | Descripción |
|-------|-------------|
| Nombre de Instancia | Nombre para tu instancia (ej. `mybot`), debe ser único en esta máquina |
| Modelo | Elige un modelo: Haiku 4.5 / Sonnet 4.5 / Opus 4.5 / Opus 4.6 |
| Canal | Elige un canal: Telegram o Feishu |

![Información básica](/images/OpenClaw-Auto/wizard-step1-basic-info.png)

### 2.2 — Credenciales y Despliegue

Todos los canales requieren una **Clave API de EvoLink**.

<Tabs>
  <Tab title="Telegram">
    Ingresa el **Bot Token** de @BotFather, luego haz clic en **Deploy**.

    El script realizará automáticamente:
    1. Crear el archivo de configuración de OpenClaw
    2. Escribir la configuración del modelo y la clave API
    3. Configurar y habilitar el plugin de Telegram
    4. Instalar e iniciar el servicio de gateway

    ![Credenciales de Telegram](/images/OpenClaw-Auto/wizard-step2-telegram-credentials.png)
  </Tab>
  <Tab title="Feishu">
    Ingresa el **App ID** y el **App Secret** de Feishu Open Platform, luego haz clic en **Deploy**.

    El script realizará automáticamente:
    1. Crear el archivo de configuración de OpenClaw
    2. Escribir la configuración del modelo y la clave API
    3. Configurar el plugin de Feishu (escribir App ID / App Secret) y habilitarlo
    4. Instalar e iniciar el servicio de gateway

    ![Credenciales de Feishu](/images/OpenClaw-Auto/wizard-step2-credentials.png)
  </Tab>
</Tabs>

Espera a que la barra de progreso se complete, luego avanzarás al siguiente paso automáticamente.

### 2.3 — Conectar

<Tabs>
  <Tab title="Telegram">
    Ingresa tu **Telegram User ID** (el ID numérico de @userinfobot), luego haz clic en **Connect**.

    El script escribe este ID en la configuración de `allowedUsers` — solo este usuario podrá chatear con el Bot. Una vez conectado, serás redirigido a la página principal.

    ![Conectar Telegram](/images/OpenClaw-Auto/wizard-step3-telegram-connect.png)
  </Tab>
  <Tab title="Feishu">
    El despliegue está completo. Ahora regresa a Feishu Open Platform para configurar la suscripción de eventos.

    <Warning>
    Antes de configurar la suscripción de eventos, asegúrate de que lo siguiente esté hecho:
    - La configuración del canal de Feishu está completa (ver despliegue del Paso 2)
    - El gateway está en ejecución (verifica con `openclaw gateway status`)
    </Warning>

    En la página de **Event Subscription**:

    1. Selecciona **Use long connection to receive events** (modo WebSocket)
    2. Agrega el evento: `im.message.receive_v1` (recibir mensajes)

    <Warning>
    Si el gateway no está iniciado o el canal no ha sido agregado, la configuración de conexión larga no se guardará.
    </Warning>

    ![Suscripción de eventos](/images/OpenClaw-Feishu/event-subscription.avif)

    3. En "Version Management & Release", crea una versión y envíala para revisión

    <Note>
    Las apps de Feishu requieren aprobación del administrador antes de poder usarse. Las apps empresariales personalizadas generalmente se aprueban automáticamente.
    </Note>

    Haz clic en **Done** — el script reiniciará el gateway para establecer la conexión WebSocket de Feishu. Después de ser redirigido a la página principal, puedes comenzar a enviar mensajes al Bot en Feishu.

    ![Conectar Feishu](/images/OpenClaw-Auto/wizard-step3-feishu-connect.png)
  </Tab>
</Tabs>

## Gestionar Instancias

Después del despliegue, la página principal muestra todas las instancias:

![Gestión de instancias](/images/OpenClaw-Auto/home-instance-running.png)

| Estado | Descripción |
|--------|-------------|
| 🟢 Verde | Gateway en ejecución |
| 🔴 Rojo | Gateway detenido |

Cada tarjeta de instancia muestra el modelo y el canal en uso. Acciones disponibles:

- **Iniciar / Detener**: Iniciar o detener el servicio de gateway
- **Eliminar**: Eliminar la instancia (detiene el gateway y limpia todos los archivos de configuración)

Haz clic en **+ New Instance** para crear múltiples instancias, cada una ejecutándose de forma independiente en un puerto diferente.
