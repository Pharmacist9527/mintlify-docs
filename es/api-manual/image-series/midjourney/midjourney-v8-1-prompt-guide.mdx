## Resumen de parámetros

| Parámetro | Sintaxis | Tipo | Rango de valores | Valor predeterminado | Descripción |
|------|------|------|---------|--------|------|
| Relación de aspecto | `--ar W:H` | Ratio entero | Cualquier ratio de enteros positivos | 1:1 | Relación de aspecto de la imagen |
| Caos | `--c N` | int | 0 - 100 | 0 | Diversidad de los resultados generados |
| Semilla | `--seed N` | int | 0 - 4294967295 | Aleatorio | Fijar semilla para reproducir resultados |
| Estilización | `--s N` | int | 0 - 1000 | 100 | Intensidad del estilo artístico |
| Parámetro experimental | `--exp N` | int | 0 - 100 | 0 | Efecto estético, acumulable con la estilización |
| Modo raw | `--raw` | Indicador | — | Desactivado | Desactivar embellecimiento predeterminado |
| Peso de imagen | `--iw N` | float | 0 - 3 | 1 | Influencia del prompt de imagen |
| Referencia de estilo | `--sref [URL]` | URL | — | — | Coincidir con el estilo visual |
| Peso de estilo | `--sw N` | int | 0 - 1000 | 100 | Intensidad de la referencia de estilo |
| Referencia de objeto | `--oref [URL]` | URL | — | — | Colocar un personaje/objeto en la escena |
| Peso de referencia de objeto | `--ow N` | int | 1 - 1000 | 100 | Intensidad de la referencia de objeto |

<Note>
Dos ajustes **no** se escriben en el prompt: se controlan a través de la API (escribirlos en el prompt no tiene efecto):

- **Velocidad** (`draft` / `fast`) → `model_params.speed`
- **Calidad de salida** (`standard` / `hd`) → parámetro `quality` de nivel superior

`--v` / `--version` está bloqueado en V8.1 y `--niji` no es compatible. Consulte [Modos de velocidad](#modos-de-velocidad) y [Calidad de salida](#calidad-de-salida) más abajo.
</Note>

## Parámetros básicos

### Relación de aspecto `--ar`

Establece la relación de aspecto de la imagen. Solo admite ratios de enteros.

```
a cat --ar 16:9
```

Valores comunes: `1:1`, `4:3`, `3:2`, `16:9`, `9:16`, `2:3`. Los ratios extremos pueden producir resultados impredecibles.

### Caos `--chaos` / `--c`

Controla la diversidad de los resultados generados. Cuanto mayor sea el valor, mayor será la diferencia entre las 4 imágenes.

```
a cat --c 50
```

| Rango | Efecto |
|------|------|
| 0 | Las 4 imágenes son muy similares (predeterminado) |
| 1-50 | Diversidad moderada |
| 50-100 | Alta aleatoriedad, ideal para exploración creativa |

### Estilización `--stylize` / `--s`

Ajusta el equilibrio entre realismo y estilo artístico.

```
a cat --s 500
```

| Rango | Efecto |
|------|------|
| 0-250 | Estilo realista, fiel al prompt |
| 250-750 | Equilibrado |
| 750-1000 | Fuerte expresión artística, colores y composición audaces |

### Parámetro experimental `--exp`

Similar a `--stylize` pero acumulable, genera imágenes más detalladas, dinámicas y creativas.

```
a cat --exp 25
```

- Valores recomendados: 5, 10, 25, 50, 100
- De 5 a 50 el cambio de efecto es notable, de 50 a 100 los rendimientos son decrecientes
- Por encima de 25-50 puede anular el efecto de `--stylize`

### Modo raw `--raw`

Desactiva el embellecimiento predeterminado y sigue más estrictamente los detalles del prompt. Adecuado para escenas que buscan realismo o control preciso.

```
product photo --raw
```

### Semilla `--seed`

Fija el estado aleatorio inicial, útil para pruebas comparativas.

```
a cat --seed 23453422
```

- Rango: 0 - 4294967295
- Misma semilla + mismo prompt → resultados similares (no idénticos)

## Parámetros de referencia de imagen

### Prompt de imagen (imagen de referencia)

Coloque las URLs de imagen al **inicio** del prompt para influir en el contenido generado.

```
https://example.com/ref.jpg text description --iw 1.5
```

**Reglas de combinación válidas:**

| Combinación | ¿Es válida? |
|---------|---------|
| 1 imagen + sin texto | ❌ No válido (producirá error) |
| 1 imagen + descripción de texto | ✅ Válido |
| 2+ imágenes + sin texto | ✅ Válido |
| 2+ imágenes + descripción de texto | ✅ Válido |

- Máximo 20 prompts de imagen
- Formatos admitidos: `.png`, `.gif`, `.webp`, `.jpg`, `.jpeg`
- No disponible con la velocidad `draft` (use `fast`)

### Peso de imagen `--iw`

Controla la influencia de los prompts de imagen en el resultado. Rango 0 - 3, predeterminado 1. Cuanto mayor sea el valor, más se acerca a la imagen de referencia.

```
https://example.com/ref.jpg watercolor landscape --iw 2.0
```

### Referencia de estilo `--sref`

Coincide con el estilo visual de la imagen de referencia (colores, texturas, iluminación), sin copiar el contenido. **Debe usarse con un prompt de texto**.

```
a cat --sref https://example.com/style.jpg
```

- Admite múltiples: `--sref URL1 URL2`
- Admite estilo aleatorio: `--sref random` (devuelve un código numérico después de la generación, reutilizable)
- Máximo 20 sref

### Peso de estilo `--sw`

Controla la influencia de la referencia de estilo. Rango 0 - 1000, predeterminado 100.

```
a cat --sref https://example.com/style.jpg --sw 500
```

### Referencia de objeto `--oref`

Coloca personajes, objetos, vehículos o criaturas no humanas de la imagen de referencia en una nueva imagen. **Debe usarse con un prompt de texto**.

```
a duckling holding a magic wand --oref https://example.com/duck.jpg
```

<Warning>
Solo se puede usar una imagen de referencia de objeto, y la generación tarda más cuando está activa.
</Warning>

### Peso de referencia de objeto `--ow`

Rango 1 - 1000, predeterminado 100. Se recomienda mantenerlo por debajo de 400, de lo contrario los resultados pueden ser impredecibles. Valores altos de `--stylize` pueden requerir un `--ow` más alto para compensar.

```
duckling --oref https://example.com/duck.jpg --ow 300
```

## Modos de velocidad

El modo de velocidad se controla mediante el campo `model_params.speed` de la API (no escriba `--draft` / `--fast` en el prompt).

| Modo | Descripción | Costo |
|------|------|------|
| `draft` | Devuelve 24 imágenes de boceto ligeras de 0.5K en una sola ejecución. Solo para la interfaz de generación de imágenes. | Mismo multiplicador que fast |
| `fast` | Modo estándar (predeterminado), 4 imágenes por generación | Estándar |

<Note>
A diferencia de V7, el modo `draft` de V8.1 **no** cuesta la mitad: usa el mismo multiplicador que `fast`. Draft devuelve 24 bocetos pequeños en una ejecución; elija los que le gusten y vuelva a ejecutarlos con calidad completa. Draft no se puede combinar con la calidad `hd`.
</Note>

## Calidad de salida

La resolución de salida se controla mediante el parámetro `quality` de nivel superior, **no** mediante un indicador en el prompt. No escriba `--hd` en el prompt.

| Valor | Resolución | Costo |
|------|------|------|
| `standard` | Resolución estándar (predeterminado) | 1x |
| `hd` | Salida HD nativa 2K | 1.5x |

- El multiplicador de calidad se **combina (multiplica)** con el multiplicador de velocidad.
- `hd` es **incompatible** con la velocidad `draft`.

---

## Dependencias

Los siguientes parámetros requieren otros parámetros para funcionar:

| Parámetro | Dependencia previa |
|------|---------|
| `--sw` | Requiere `--sref` |
| `--ow` | Requiere `--oref` |
| `--iw` | Requiere prompt de imagen |
| `--sref`, `--oref` | Requiere prompt de texto |
| Prompt de imagen único | Requiere prompt de texto |

---

## Conflictos

| Parámetro A | Parámetro B | Descripción |
|--------|--------|------|
| Velocidad `draft` | Calidad `hd` | El modo borrador no se puede combinar con HD |
| Velocidad `draft` | Prompt de imagen / `--sref` / `--oref` | Las referencias de imagen requieren `fast` |
| Prompt solo de imagen | `--stylize` / `--exp` | Incompatible sin texto |
| `--exp` > 25 | `--stylize` | Un `--exp` alto puede suprimir `--stylize`; reduzca `--exp` al combinarlos |

## Parámetros no admitidos en V8.1

| Parámetro | Descripción |
|------|------|
| `--q` / `--quality` | Indicador de nivel de detalle no expuesto en esta ruta (use el parámetro `quality` de nivel superior para la resolución) |
| `--no` | Prompt negativo no expuesto en esta ruta |
| `--weird` / `--w` | No admitido |
| `--tile` | No admitido |
| `--sv` | La versión de estilo está bloqueada; no expuesta |
| `--stop` | No admitido |
| `--cref` / `--cw` | Use `--oref` referencia de objeto en su lugar |
| `--relax` | No admitido |
| `--repeat` / `--r` | No admitido a través de la API |
| `--p` (personalize) | No admitido a través de la API |
| `{}` enumeración | No admitido a través de la API |
| `--stealth` / `--public` | No compatible con la API (las imágenes no son de acceso público) |
| `--niji` | Es un modelo diferente, no V8.1 |
| Multi-prompt `::` | No admitido |

## Reglas de formato de prompt

### Estructura básica

```
[image URL] text description --param1 value1 --param2 value2
```

### Reglas de escritura

- Los parámetros van al **final** del prompt de texto
- Debe haber un **espacio** antes de `--`
- No usar signos de puntuación dentro de los parámetros
- No escribir texto después de los parámetros

### Ejemplos

<CodeGroup>
```text Correcto
a yellow Persian cat --ar 1:2 --s 500
```

```text Incorrecto
a yellow Persian cat--ar 2:3          ← missing space before --
a yellow Persian cat - - ar 2:3       ← space between --
a yellow Persian cat --ar 2:3, --s 50 ← punctuation in parameters
a yellow Persian cat --ar 2:3 more description ← text after parameters
```
</CodeGroup>

## Aviso de moderación de contenido

<Note>
Midjourney cuenta con un sistema de moderación de contenido integrado. Si algunas imágenes generadas son filtradas por la moderación, los créditos consumidos en esa solicitud no serán reembolsados. Asegúrese de que sus prompts cumplan con las directrices de contenido.
</Note>
