## Resumen de parámetros

| Parámetro | Sintaxis | Tipo | Rango de valores | Valor predeterminado | Descripción |
|------|------|------|---------|--------|------|
| Relación de aspecto | `--ar W:H` | Ratio entero | Cualquier ratio de enteros positivos | 1:1 | Relación de aspecto de la imagen |
| Caos | `--c N` | int | 0 - 100 | 0 | Diversidad de los resultados generados |
| Prompt negativo | `--no palabra1,palabra2` | Palabras clave | — | — | Excluir elementos no deseados |
| Calidad | `--q N` | Enumeración | 1, 2, 4 | 1 | Nivel de detalle, **afecta el costo** |
| Semilla | `--seed N` | int | 0 - 4294967295 | Aleatorio | Fijar semilla para reproducir resultados |
| Estilización | `--s N` | int | 0 - 1000 | 100 | Intensidad del estilo artístico |
| Weird | `--w N` | int | 0 - 3000 | 0 | Grado de lo inusual/surrealista |
| Parámetro experimental | `--exp N` | int | 0 - 100 | 0 | Efecto estético (exclusivo de V7) |
| Modo raw | `--raw` | Indicador | — | Desactivado | Desactivar embellecimiento predeterminado |
| Mosaico | `--tile` | Indicador | — | Desactivado | Patrón repetible sin costuras |
| Peso de imagen | `--iw N` | float | 0 - 3 | 1 | Influencia de la imagen de referencia |
| Referencia de estilo | `--sref [URL]` | URL | — | — | Coincidir con el estilo visual |
| Peso de estilo | `--sw N` | int | 0 - 1000 | 100 | Intensidad de la referencia de estilo |
| Versión de estilo | `--sv N` | enumeración | 1, 2, 3, 4, 5, 6 | 4 | Versión del algoritmo de estilo |
| Referencia de objeto | `--oref [URL]` | URL | — | — | Colocar personajes/objetos (exclusivo de V7), **costo doble** |
| Peso de referencia de objeto | `--ow N` | int | 1 - 1000 | 100 | Intensidad de la referencia de objeto |
| Modo borrador | `--draft` | bandera | — | desactivado | 10x más rápido, mitad de costo (exclusivo V7), controlado por API `model_params.speed` |

<Note>
El modo de velocidad (draft / fast / turbo) se controla mediante el campo `model_params.speed` de la API. No es necesario escribir `--draft` / `--fast` / `--turbo` manualmente en el prompt.
</Note>

## Parámetros básicos

### Relación de aspecto `--ar`

Establece la relación de aspecto de la imagen. Solo admite ratios de enteros.

```
un gato --ar 16:9
```

Valores comunes: `1:1`, `4:3`, `3:2`, `16:9`, `9:16`, `2:3`

### Caos `--chaos` / `--c`

Controla la diversidad de los resultados generados. Cuanto mayor sea el valor, mayor será la diferencia entre las 4 imágenes.

```
un gato --c 50
```

| Rango | Efecto |
|------|------|
| 0 | Las 4 imágenes son muy similares (predeterminado) |
| 1-50 | Diversidad moderada |
| 50-100 | Alta aleatoriedad, ideal para exploración creativa |

### Prompt negativo `--no`

Excluye elementos no deseados, equivalente a un peso de -0.5. Admite múltiples palabras clave.

```
campo vibrante de tulipanes --no rojo,sombras
```

<Warning>
V7 no admite la sintaxis de multi-prompt `::`. Solo se puede usar `--no` para excluir elementos no deseados.
</Warning>

### Calidad `--quality` / `--q`

Controla el nivel de detalle de la generación.

```
un gato --q 2
```

| Valor | Descripción |
|----|------|
| 1 | Predeterminado, coherencia de manos optimizada |
| 2 | Usa el modelo V7 pre-optimización |
| 4 | Modo experimental (incompatible con `--oref`) |

<Info>
Cuanto mayor sea el valor de calidad, mayor será el consumo de GPU y el costo correspondiente.
</Info>

### Semilla `--seed`

Fija el estado aleatorio inicial, útil para pruebas comparativas.

```
un gato --seed 23453422
```

- Rango: 0 - 4294967295
- Misma semilla + mismo prompt → resultados similares (no idénticos)
- No aplicable al modo de velocidad turbo

### Estilización `--stylize` / `--s`

Ajusta el equilibrio entre realismo y arte.

```
un gato --s 500
```

| Rango | Efecto |
|------|------|
| 0-250 | Estilo realista, fiel al prompt |
| 250-750 | Equilibrado |
| 750-1000 | Fuerte artisticidad, colores y composición audaces |

### Weird `--weird` / `--w`

Introduce elementos inusuales y surrealistas.

```
paisaje espacial --w 800
```

| Rango | Efecto |
|------|------|
| 10-300 | Ligeramente extraño |
| 300-1000 | Elementos claramente no tradicionales |
| 1000-3000 | Surrealismo y abstracción extremos |

### Parámetro experimental `--exp` (exclusivo de V7)

Similar a `--stylize` pero acumulable, genera imágenes más detalladas, dinámicas y creativas.

```
un gato --exp 25
```

- Valores recomendados: 5, 10, 25, 50, 100
- De 5 a 50 el cambio de efecto es notable, de 50 a 100 el cambio es menor
- Por encima de 25-50 puede anular el efecto de `--stylize`

### Modo raw `--raw`

Desactiva el embellecimiento predeterminado y sigue más estrictamente los detalles del prompt. Adecuado para escenas que buscan realismo o control preciso.

```
foto de producto --raw
```

### Mosaico `--tile`

Genera patrones repetibles sin costuras (conexión perfecta en las 4 direcciones), adecuado para texturas, fondos de pantalla y diseño de telas.

```
textura de pared de ladrillos --tile
```

## Parámetros de referencia de imagen

### Prompt de imagen (imagen de referencia)

Coloque la URL de la imagen al **inicio** del prompt para influir en el contenido generado con la imagen.

```
https://example.com/ref.jpg descripción de texto --iw 1.5
```

**Reglas de combinación válidas:**

| Combinación | ¿Es válida? |
|---------|---------|
| 1 imagen + sin texto | ❌ No válido (producirá error) |
| 1 imagen + descripción de texto | ✅ Válido |
| 2+ imágenes + sin texto | ✅ Válido |
| 2+ imágenes + descripción de texto | ✅ Válido |

- Máximo 20 imágenes de referencia
- Formatos admitidos: `.png`, `.gif`, `.webp`, `.jpg`, `.jpeg`

### Peso de imagen `--iw`

Controla la influencia de la imagen de referencia en el resultado. Rango 0 - 3, predeterminado 1, cuanto mayor sea el valor, más se acerca a la imagen de referencia.

```
https://example.com/ref.jpg paisaje de acuarela --iw 2.0
```

### Referencia de estilo `--sref`

Coincide con el estilo visual de la imagen de referencia (colores, texturas, iluminación), sin copiar el contenido. **Debe usarse con un prompt de texto**.

```
un gato --sref https://example.com/style.jpg
```

- Admite múltiples: `--sref URL1 URL2`
- Admite asignación de pesos: `--sref URL1::2 URL2::1` (Nota: el `::` aquí es la sintaxis de peso dentro del parámetro sref, un concepto diferente del separador multi-prompt `::` que V7 no soporta)
- Admite estilo aleatorio: `--sref random` (devuelve un código numérico después de la generación, reutilizable)
- Máximo 20 sref

### Peso de estilo `--sw`

Controla el grado de influencia de la referencia de estilo. Rango 0 - 1000, predeterminado 100.

```
un gato --sref https://example.com/style.jpg --sw 500
```

### Versión de estilo `--sv`

Cambia la versión del algoritmo de referencia de estilo.

| Versión | Características |
|------|------|
| 1 | Algoritmo original, resultado más "atmosférico" |
| 2 | Segunda iteración |
| 3 | Versión actualizada de v1 |
| 4 | Versión actualizada de v2 (**predeterminado**) |
| 5 | Nuevo en V7 |
| 6 | Nuevo en V7 |

### Referencia de objeto `--oref` (exclusivo de V7)

Coloca personajes, objetos, vehículos o criaturas no humanas de la imagen de referencia en una nueva imagen. **Debe usarse con un prompt de texto**.

```
un patito con una varita mágica --oref https://example.com/duck.jpg
```

<Warning>
El uso de referencia de objeto consume el doble de tiempo de GPU y duplica el costo. Además, solo se puede usar una imagen de referencia.
</Warning>

### Peso de referencia de objeto `--ow` (exclusivo de V7)

Rango 1 - 1000, predeterminado 100. Se recomienda mantenerlo por debajo de 400, de lo contrario los resultados pueden ser impredecibles. Valores altos de `--stylize` pueden requerir un `--ow` más alto para compensar.

```
patito --oref https://example.com/duck.jpg --ow 300
```

## Modos de velocidad

El modo de velocidad se controla mediante el campo `model_params.speed` de la API, y el sistema lo inyecta automáticamente en el prompt.

| Modo | Descripción | Costo |
|------|------|------|
| `draft` | 10 veces más rápido, menor calidad, se puede mejorar posteriormente con [mj-v7-enhance](/es/api-manual/image-series/midjourney/mj-v7-enhance). **Solo para la interfaz de generación de imágenes** | Mínimo |
| `fast` | Calidad estándar (predeterminado) | Estándar |
| `turbo` | Velocidad máxima | Aproximadamente el doble de fast |

---

## Dependencias

Los siguientes parámetros requieren otros parámetros para funcionar:

| Parámetro | Dependencia previa |
|------|---------|
| `--sv`, `--sw` | Requiere `--sref` |
| `--iw` | Requiere prompt de imagen (imagen de referencia) |
| `--ow` | Requiere `--oref` |
| `--sref`, `--oref` | Requiere prompt de texto |
| Imagen de referencia única | Requiere prompt de texto |

---

## Conflictos

| Parámetro A | Parámetro B | Descripción |
|--------|--------|------|
| `--oref` | Velocidad draft / turbo | Referencia de objeto solo es compatible con modo fast |
| `--oref` | `--q 4` | Calidad 4 es incompatible con referencia de objeto |
| `--oref` | inpaint/pan/outpaint | Las imágenes de referencia de objeto no se pueden usar para estas operaciones |
| `--draft` | `--tile` | El modo borrador no admite mosaico |
| Prompt solo de imagen | `--stylize` / `--weird` | Incompatible sin texto |

<Tip>
Cuando el valor de `--exp` supera 25-50, suprime el efecto de `--stylize`. Se recomienda usar un valor bajo de exp (5-25) cuando se combinan ambos.
</Tip>

---

## Parámetros no admitidos en V7

| Parámetro | Descripción |
|------|------|
| `--stop` | No admitido en V7 |
| `--cref` / `--cw` | Use `--oref` referencia de objeto en su lugar |
| `--relax` | V7 no admite modo lento |
| `--repeat` / `--r` | No admitido en la API |
| `--p` (personalize) | No admitido en la API |
| `{}` enumeración | Prompts de permutación no admitidos en la API |
| `--stealth` / `--public` | No compatible con la API (las imágenes no son de acceso público) |
| Multi-prompt `::` | No admitido en V7 |

## Reglas de formato de prompt

### Estructura básica

```
[URL de imagen] descripción de texto --parámetro1 valor1 --parámetro2 valor2
```

### Reglas de escritura

- Los parámetros van al **final** del prompt de texto
- Debe haber un **espacio** antes de `--`
- No usar signos de puntuación dentro de los parámetros
- No escribir texto después de los parámetros

### Ejemplos

<CodeGroup>
```text Correcto
un gato persa amarillo --ar 1:2 --s 500
```

```text Incorrecto
un gato persa amarillo--ar 2:3          ← falta espacio antes de --
un gato persa amarillo - - ar 2:3       ← hay espacio entre --
un gato persa amarillo --ar 2:3, --s 50 ← puntuación en los parámetros
un gato persa amarillo --ar 2:3 más descripción ← texto después de los parámetros
```
</CodeGroup>
